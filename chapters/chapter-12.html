1) Introduction

Imagine you collect unique usernames from multiple sources. You don’t want duplicates.

That is what a set is for.

2) Theory

2.1 What is a set?

A set is an unordered collection of unique items. If you add the same item twice, the set keeps only one.

2.2 Why is order not important?

Sets are optimized for membership tests. That means their output order can differ from insertion order.

⚠ Warning: sets cannot contain mutable items

Set elements must be hashable. For example, you cannot put a list inside a set.

2.3 Set operations

  • Union (A ∪ B): items in A or B.
  • Intersection (A ∩ B): items in both.
  • Difference (A − B): items in A not in B.
  • Symmetric difference: items in A or B but not both.

3) Syntax

Python
create sets
s1 = {1, 2, 2, 3}
print(s1)  # duplicates removed

empty = set()
print(type(empty))

s2 = set(["a", "b"])
print(s2)
Python
membership
users = {"ravi", "asha"}

print("ravi" in users)   # True
print("kiran" in users)  # False

Time complexity: membership tests are fast (conceptually ~O(1) average) • Space: depends on set size.

Alternative: use lists for ordered duplicates (but slower membership).

4) Examples

4.1 Easy Example — unique letters

Python
unique from string

4.2 Medium Example — union and intersection

Python
union / intersection
A = {1, 2, 3}
B = {3, 4}

print("Union:", A | B)
print("Intersection:", A & B)

Output: Union has 1,2,3,4. Intersection has only 3.

4.3 Advanced Example — difference

Python
difference
A = {"apple", "banana", "mango"}
B = {"banana", "grape"}

print("A-B:", A - B)

Output: {"apple", "mango"} (order may vary).

9) Methods

Common set methods

  • add(x): add element.
  • remove(x): remove element (error if missing).
  • discard(x): remove if exists (no error if missing).
  • pop(): remove and return an arbitrary element.
Python
add/remove
skills = {"python", "sql"}
skills.add("ml")
print(skills)

skills.discard("sql")
print(skills)

14) Mini Project

🧪 Mini Project: Unique Interest Finder

Given three comma-separated interest strings, build sets and print:

  • All unique interests (union)
  • Common interests across all three (intersection)
Python
starter
s1 = "music, sports, travel"
s2 = "music, coding, travel"
s3 = "coding, travel, movies"

A = set([x.strip() for x in s1.split(",")])
B = set([x.strip() for x in s2.split(",")])
C = set([x.strip() for x in s3.split(",")])

all_unique = A | B | C
common = A & B & C

print("All unique:", all_unique)
print("Common:", common)

16) Common Errors

  • TypeError: unhashable type (e.g., list added to set).
  • KeyError with remove() if element doesn’t exist.
  • Logic error: assuming set order is fixed.

17) Best Practices

  • Use sets for uniqueness and membership checks.
  • Use discard() for safe removals.
  • Don’t rely on printed order of sets.
  • Convert lists to sets to remove duplicates quickly.
  • Ensure elements are hashable (use tuples for pairs).

18) Summary

Sets are unique collections optimized for membership checks. You create sets with {} or set(). You can add/remove elements and use union (|), intersection (&), and difference (-). Sets are unordered and cannot contain unhashable (like lists).
