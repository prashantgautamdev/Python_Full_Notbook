1) Introduction

A website server must respond when a browser requests a URL. Flask makes it easy to write these responses in Python.

In this chapter, you build your first Flask app and learn the idea of routes.

2) Theory

2.1 Flask at a glance

Flask is a small framework that provides: routing, request handling, and response building.

2.2 Routing

Routing means: URL path → Python function. Example: / → show homepage.

2.3 Requests and responses

The browser sends a request. Flask reads request data, then returns a response (HTML, JSON, or other types).

⚠ Warning

Do not run Flask in production mode using the debug server. For learning it’s fine.

3) Syntax

Python
minimal Flask app
from flask import Flask

app = Flask(__name__)

@app.route("/")
def home():
    return "Hello from Flask!"

if __name__ == "__main__":
    app.run(host="127.0.0.1", port=5005, debug=True)

Output: start server, open http://127.0.0.1:5005/.

Time: O(1) for this tiny handler • Space: O(1).

Alternative: use Django for a heavier framework (next chapters).

4) Examples

4.1 Easy Example — route with a dynamic part

Python
/hello/ 
from flask import Flask

app = Flask(__name__)

@app.route("/hello/")
def say_hello(name):
    # name comes from URL
    return f"Hello, {name}!"

Output: /hello/Asha → Hello, Asha!

4.2 Medium Example — JSON response (concept)

Python
return dict as JSON
from flask import Flask, jsonify

app = Flask(__name__)

@app.route("/api/version")
def version():
    payload = {"version": "1.0.0"}
    return jsonify(payload)

Output: JSON like {"version":"1.0.0"}

4.3 Advanced Example — reading query parameters (concept)

Python
?name=...
from flask import Flask, request

app = Flask(__name__)

@app.route("/welcome")
def welcome():
    name = request.args.get("name", "Guest")
    return f"Welcome {name}!"

Output: /welcome?name=Ravi → greeting.

6) Memory Visualization

When a request arrives, Flask holds request data while executing the handler. After response is returned, temporary objects are released.

Part Purpose
Request object stores query/form data
Handler function computes response payload
Response object stores status + body

7) Dry Run

Dry run
GET /welcome?name=Asha
Step 1: Flask receives request for /welcome
Step 2: route matches /welcome ✅
Step 3: request.args.get("name") gives "Asha"
Step 4: handler returns "Welcome Asha!"
Step 5: Flask sends response to browser

Time: O(1) • Space: O(1) for small strings.

15) Debugging Practice

Case 1: route not found (404)

bug
wrong URL
# app route is /hello/
# but you open /hell0/Asha (typo)

Fix: confirm exact route spelling, and check trailing slashes.

Case 2: JSON parsing error

bug
wrong response type
return {"x": 1}
# without jsonify

Fix: use jsonify(payload) or return JSON properly.

Case 3: crash on invalid query param

bug
int conversion
a = int(request.args.get("a"))
# a might be missing or not a number

Fix: wrap in try/except, return 400 with error JSON.

16) Common Errors

  • 404: wrong path or missing route.
  • 500: exception inside handler (ValueError, KeyError, etc.).
  • TypeError: wrong return type or wrong function usage.
  • JSON error: returning dict without jsonify (concept).

17) Best Practices

  • Use small handlers and helper functions.
  • Validate all user input.
  • Use consistent JSON shapes for API responses.
  • Return meaningful status codes (200/201/400/404/500).
  • Don’t leak internal errors to clients.

Key explanations

  • 404: route/path not found.
