1) Introduction

When you use an app (maps, shopping, weather), the app does not compute everything itself. It talks to a server using APIs.

In this chapter, you learn how APIs work at a beginner-friendly level. Later chapters (Flask and Django) will help you build real APIs.

2) Theory

2.1 What is an API?

API = Application Programming Interface. It defines how clients can request data and actions.

2.2 HTTP methods (the verbs)

  • GET: read data
  • POST: create data
  • PUT: replace/update fully
  • PATCH: partial update
  • DELETE: delete data

2.3 Status codes (response meaning)

Code Meaning When it happens
200 OK Request succeeded
201 Created A POST created a resource
400 Bad Request Client sent wrong input
404 Not Found No matching endpoint/data
500 Server Error Bug on server side

2.4 JSON and structured responses (concept)

APIs usually return JSON. JSON is simple: key → value pairs.

⚠ Warning

Never trust client input. Validate and sanitize everything.

3) Syntax

Concept
endpoint definition
GET /api/hello
→ returns {"message": "Hello"}

POST /api/users
(body: {"name": "Asha"})
→ creates user and returns {"id": 1}
Python
example using http.server (concept)
# This is a minimal concept example.
# For production APIs you usually use Flask/Django/FastAPI.

from http.server import BaseHTTPRequestHandler, HTTPServer
import json

class Handler(BaseHTTPRequestHandler):
    def do_GET(self):
        # Decide route using the path
        if self.path == "/api/hello":
            payload = {"message": "Hello from API"}
            body = json.dumps(payload).encode("utf-8")

            self.send_response(200)  # status code
            self.send_header("Content-Type", "application/json")
            self.send_header("Content-Length", str(len(body)))
            self.end_headers()

            # Output the response body
            self.wfile.write(body)
        else:
            # Unknown endpoint
            self.send_response(404)
            self.end_headers()

server = HTTPServer(("127.0.0.1", 5004), Handler)
print("API running on http://127.0.0.1:5004")
server.serve_forever()

Output: When you open /api/hello, you receive JSON like {"message":"Hello from API"}.

Time: O(1) per request (for this simple route) • Space: O(response_size).

Alternative: use requests for clients and Flask/Django for real servers.

4.2 Medium Example — input validation (concept)

If a client sends invalid data, return 400.

Python
validation idea
# concept: validate name is not empty

def validate_user_payload(payload):
    # payload is a dict
    name = payload.get("name")
    if not isinstance(name, str):
        return False, "name must be a string"
    if not name.strip():
        return False, "name cannot be empty"

    return True, "ok"

15) Debugging Practice

Case 1: wrong path

bug
returns 404
if self.path == "/api/hello":
    ...
else:
    404

Fix: make sure the client calls exactly /api/hello (no trailing spaces, correct leading slash).

Case 2: wrong Content-Type

bug
client can’t parse JSON
self.send_header("Content-Type", "text/plain")

Fix: set application/json so clients understand the format.

Case 3: missing Content-Length

bug
client may wait
# forgot Content-Length

Fix: compute and send Content-Length for proper response framing (especially with some clients).

16) Common Errors

  • 400 Bad Request: invalid payload or missing keys.
  • 404 Not Found: wrong endpoint path.
  • 415 Unsupported Media Type (concept): wrong Content-Type sent by client.
  • 500 Internal Server Error: server crash due to bug.

17) Best Practices

  • Design clear endpoint names and stable response formats.
  • Validate input and return correct status codes.
  • Use consistent error objects.
  • Set Content-Type and (when needed) Content-Length.
  • Keep handlers small: use helper functions for logic.

22) Answer Key (Explain every answer)

Key explanations

  • API: a contract between client and server for data/actions.
  • Endpoint: path + method where server handles requests.
  • GET: read data without creating/updating.
  • POST: create new resource or submit data.
  • 404: the requested path (or data) was not found.
  • Validate: prevents wrong data and reduces security issues.
  • JSON: structured key-value format widely used for APIs.
