1) Introduction

Text looks simple to humans. But computers need numbers.

NLP is the field that turns text into numeric features so ML can learn from it.

2) Theory

2.1 Tokenization

Tokenization means splitting text into pieces called tokens. Often tokens are words.

2.2 Preprocessing

Common preprocessing steps:

  • Lowercase text
  • Remove punctuation (concept)
  • Split into tokens

2.3 Bag-of-Words (BoW) concept

BoW makes a vocabulary of all words. Then each sentence becomes a vector where each position is a word count.

⚠ Warning

BoW loses word order. “I love dogs” and “dogs love I” can look similar.

2.4 Simple classification concept

Once text is converted into vectors, you can use ML algorithms. We will build a tiny nearest-centroid / majority approach for explanation.

3) Syntax

Python
tokenize + bag-of-words vector
import re
from collections import Counter


def preprocess(text):
    # Lowercase for consistency
    text = text.lower()
    # Keep letters/numbers/spaces only (simple concept)
    text = re.sub(r"[^a-z0-9\s]", " ", text)
    # Split into tokens
    tokens = text.split()
    return tokens


def bow_vector(tokens, vocab):
    # vocab is a list of known words
    counts = Counter(tokens)
    # Create vector in vocab order
    return [counts.get(word, 0) for word in vocab]


text = "I love dogs!"
print(preprocess(text))

Time: O(n) for tokenization + O(|vocab|) for vector • Space: O(|vocab|).

Alternative: use TF-IDF (later concept) for better weighting.

4) Examples

4.1 Easy Example — vocabulary + vectors

Python
BoW for two sentences
import re
from collections import Counter


def preprocess(text):
    text = text.lower()
    text = re.sub(r"[^a-z0-9\s]", " ", text)
    return text.split()


sent1 = "I love dogs"
sent2 = "I hate cats"

t1 = preprocess(sent1)
t2 = preprocess(sent2)

vocab = sorted(set(t1 + t2))

v1 = [Counter(t1).get(w,0) for w in vocab]
v2 = [Counter(t2).get(w,0) for w in vocab]

print("vocab:", vocab)
print("v1:", v1)
print("v2:", v2)

Output: a vocab list and two integer count vectors.

4.2 Medium Example — tiny rule-based classifier

Python
keyword counts (concept)
import re


def preprocess(text):
    text = text.lower()
    text = re.sub(r"[^a-z0-9\s]", " ", text)
    return text.split()


def predict_sentiment(text):
    tokens = preprocess(text)

    positive_words = {"love", "great", "awesome"}
    negative_words = {"hate", "bad", "awful"}

    pos = sum(1 for t in tokens if t in positive_words)
    neg = sum(1 for t in tokens if t in negative_words)

    if pos > neg:
        return "positive"
    if neg > pos:
        return "negative"
    return "neutral"


print(predict_sentiment("I love this"))
print(predict_sentiment("I hate that"))

Output: positive/negative/neutral labels.

Flow idea

16) Common Errors

  • ValueError: vector length mismatch.
  • KeyError: missing word in vocab mapping.
  • Empty tokens: preprocessing removed everything.
  • TypeError: passing string where tokens list expected.

17) Best Practices

  • Keep preprocessing consistent across train/test.
  • Build vocab on training data only (avoid leakage).
  • Start with simple BoW to understand the pipeline.
  • Use evaluation metrics appropriate for your dataset (concept).
  • Track vocabulary size and performance.

22) Answer Key (Explain every answer)

Key explanations

  • NLP: makes computers understand/operate on human language.
  • Tokenization: splits text into pieces (often words).
  • Preprocessing: normalizes text so tokens are consistent.
  • BoW: counts words into a fixed-length vector.
  • Word order: BoW does not keep order (disadvantage).
  • Vocabulary: list of known words used as vector positions.
  • Vectors: numeric representations used by ML.
  • Split: test data must stay unseen to estimate real quality.
