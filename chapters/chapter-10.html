Learn to store many values in one variable using Python lists: create, index, slice, update, and use common list methods.

🎯 Learning Objectives

By the end of this chapter, you will be able to:

1) Introduction

In earlier chapters, you stored one value in one variable. But real data often has many values.

A list lets you store many items in one variable. You can access items by position.

📌 Real-life example

Think of a shopping cart: many products inside one cart. The cart is the list, and products are items.

2) Theory

2.1 What is a list?

A list is an ordered collection. Items have an index starting at 0.

2.2 Lists are mutable

“Mutable” means you can change the list after creation. For example, you can replace an item or add new items.

⚠ Warning

If you try to access an index that doesn’t exist, Python raises IndexError.

2.3 Lists vs strings

Feature List String
Editable? Yes (mutable) No (immutable)
Item type Can be different types Usually text characters
Indexing Yes Yes

3) Syntax

Python
create and access
numbers = [10, 20, 30]
print(numbers[0])  # 10
print(numbers[-1]) # 30
Python
slicing
nums = [1, 2, 3, 4, 5]
print(nums[1:4])  # [2,3,4]
print(nums[:2])   # [1,2]
print(nums[3:])   # [4,5]
Python
modify
nums = [1, 2, 3]
nums[1] = 99
print(nums)

Time complexity: indexing is O(1), slicing is O(k) • Space: slicing creates a new list O(k).

Alternative: use loops to process lists without slicing if memory matters.

4) Examples

4.1 Easy Example — add items with append()

Python
append
colors = ["red", "green"]
colors.append("blue")
print(colors)

Output: ['red', 'green', 'blue']

4.2 Medium Example — replace and sum

Python
update + compute
prices = [5, 10, 15]
prices[0] = 8

total = 0
for p in prices:
    total += p

print("Total:", total)

Output: Total: 33

Time: O(n) • Space: O(1) extra.

4.3 Advanced Example — list as mixed types

Python
mixed list
data = ["Alice", 21, True]
print(data)
print(data[0])
print(data[2])

Output: shows items, then first and boolean.

9) Methods

Common list methods

  • append(x): add one item at the end.
  • pop(): remove and return the last item.
  • sort(): sort list in place (concept here).
Python
append + pop
stack = ["a", "b"]
stack.append("c")
print(stack)

removed = stack.pop()
print("Removed:", removed)
print(stack)

Output: stack grows then shrinks.

15) Debugging Practice

Case 1: IndexError

Python
bug
nums = [1, 2]
print(nums[2])

Fix: valid indices are 0..1. Use nums[1] or check length.

Case 2: trying to edit string

Python
bug
s = "cat"
s[0] = "b"
print(s)

Fix: strings are immutable; use lists if you need modification.

Case 3: wrong loop indentation

Python
bug
nums = [1,2,3]
for n in nums:
print(n)

Fix: indent print(n) under the loop.

16) Common Errors

  • IndexError: using an index that doesn’t exist.
  • TypeError: mixing list and non-list operations.
  • SyntaxError: missing brackets or commas.

17) Best Practices

  • Use clear variable names: items, , .
  • Check len(list) before indexing near the end.
  • Prefer loops for processing many items.
  • Use slicing for safe sub-parts.
  • Keep code formatting and indentation clean.

18) Summary

Lists store multiple ordered items. You can access items using indexing and slicing. Lists are mutable so you can update elements and use methods like append() and pop().

22) Answer Key (Explain every answer)

Key explanations

  • List created with: [ ].
  • Indexing starts at 0: first item is position 0.
  • Negative -1: last item.
  • Slicing excludes end: [start:end] includes start but excludes end.
  • append adds at end: increases list size by 1.
  • pop removes last: returns removed item.
