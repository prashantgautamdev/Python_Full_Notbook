1) Introduction

Machine learning learns patterns. Deep learning goes further by using multiple layers of neurons.

Each layer learns a simpler pattern. Together they form a complex idea.

2) Theory

2.1 Neuron basics

A neuron takes inputs, multiplies by weights, adds a bias, then applies an activation.

⭐ Remember

Activation is like a “smart squashing” function. It lets networks learn non-linear patterns.

2.2 Layers and deep networks

  • Input layer: receives data.
  • Hidden layers: learn features.
  • Output layer: produces predictions.

2.3 Loss and training (concept)

Training means: make predictions → compute loss → update weights to reduce loss.

⚠ Warning

Deep learning is powerful, but training can overfit. Always evaluate on validation/test data.

3) Syntax

Python
tiny forward pass (concept)
import math

# Activation function: sigmoid (concept)
def sigmoid(x):
    return 1 / (1 + math.exp(-x))

# A single neuron forward pass
# y = sigmoid(w1*x1 + w2*x2 + b)
w1 = 0.5
w2 = -1.2
b = 0.3

x1 = 2
x2 = 1

z = w1*x1 + w2*x2 + b
y = sigmoid(z)

print("z:", z)
print("y:", y)

# Output: depends on your numbers.

Output: shows intermediate value z and final activation y.

Time: O(1) • Space: O(1).

Alternative: use ReLU activation for simpler math in practice.

4.2 Medium Example — forward pass for 2 inputs

Python
manual compute
import math

def sigmoid(x):
    return 1 / (1 + math.exp(-x))

# Example weights
w = [1.0, -0.5]
b = -0.1

# Example inputs
x = [2.0, 3.0]

# Compute z = sum(w_i * x_i) + b
z = w[0]*x[0] + w[1]*x[1] + b

# Apply activation
out = sigmoid(z)

print("z =", z)
print("output =", out)

Output: prints a number between 0 and 1 for sigmoid.

4.3 Advanced Example — tiny training loop concept

Flow idea

Repeat for epochs:

  • Forward pass → predict
  • Loss = difference from target
  • Update weights (gradient step concept)

7) Dry Run

Dry run
single neuron
Given:
  w1=0.5, w2=-1.2, b=0.3
  x1=2, x2=1

Compute z:
  z = 0.5*2 + (-1.2)*1 + 0.3
  z = 1.0 - 1.2 + 0.3
  z = 0.1

Apply sigmoid:
  y = sigmoid(0.1) ≈ 0.525

So output is about 0.525.

Time: O(1) • Space: O(1).

15) Debugging Practice

Case 1: sigmoid overflow

bug
exp too large
def sigmoid(x):
    return 1/(1+math.exp(-x))
# if x is huge, exp can overflow

Fix: use a safer sigmoid approach (concept) or clamp x.

Case 2: forgetting bias

bug
missing + b
z = w1*x1 + w2*x2
# forgot bias term b

Fix: ensure z includes bias.

Case 3: loss increases during training

bug
wrong gradient sign
# weights updated in wrong direction

Fix: verify gradient math and update rule.

16) Common Errors

  • OverflowError: sigmoid exponent too large (concept).
  • ZeroDivisionError: not typical here, but can happen in custom losses.
  • ValueError: incorrect input shapes for your manual model.
  • Logic bug: using wrong activation for output layer.

17) Best Practices

  • Start with tiny models and verify outputs by hand.
  • Print intermediate values z, y, and loss while debugging.
  • Use clear loss definitions and consistent target formats.
  • Track loss over epochs to confirm training is working.
  • Evaluate on unseen data to avoid overfitting.
