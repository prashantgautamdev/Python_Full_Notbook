1) Introduction

Artificial Intelligence (AI) is the area of computer science that builds systems that can reason and decide.

In this chapter, we focus on classic AI ideas: agents, search, and planning.

2) Theory

2.1 AI vs automation

Automation follows fixed rules. AI often learns patterns or uses reasoning to choose actions.

2.2 The agent model

An AI agent observes the environment and chooses actions.

  • Environment: the world state (maze, game, tasks).
  • Agent: the decision-making program.
  • Goal: a target the agent tries to reach.

2.3 Search

Search tries many possibilities to find a path from start to goal.

2.4 Planning (concept)

Planning chooses a sequence of actions. If an action leads to a dead end, the planner tries another sequence.

⚠ Warning

Search can become slow if you explore too many states. That’s why heuristics (later concept) help.

3) Syntax

Python
BFS concept (queue)
from collections import deque

def bfs(graph, start, goal):
    # queue holds (node, path)
    q = deque()
    q.append((start, [start]))

    # visited prevents repeating states
    visited = {start}

    while q:
        node, path = q.popleft()
        if node == goal:
            return path

        for nxt in graph.get(node, []):
            if nxt not in visited:
                visited.add(nxt)
                q.append((nxt, path + [nxt]))

    return None

Time: depends on number of nodes and edges • Space: depends on visited/path.

Alternative: use DFS when you want deep exploration (not always shortest path).

4) Examples

4.1 Easy Example — BFS shortest path in a graph

Python
graph + bfs
from collections import deque

def bfs_path(graph, start, goal):
    q = deque([(start, [start])])
    visited = {start}

    while q:
        node, path = q.popleft()
        if node == goal:
            return path

        for nxt in graph.get(node, []):
            if nxt not in visited:
                visited.add(nxt)
                q.append((nxt, path + [nxt]))

    return None

graph = {
    "A": ["B", "C"],
    "B": ["D"],
    "C": ["E"],
    "D": ["F"],
    "E": ["F"],
    "F": []
}

print(bfs_path(graph, "A", "F"))

Output: a path list like ['A','B','D','F'] (depends on graph order).

7) Dry Run

Dry run
BFS A→F
Queue start: [(A, [A])]
Visited: {A}

Pop A:
- goal? no
- add neighbors B and C
Queue: [(B,[A,B]), (C,[A,C])]

Pop B:
- goal? no
- add D
Queue: [(C,[A,C]), (D,[A,B,D])]

Pop C:
- add E
Queue: [(D,[A,B,D]), (E,[A,C,E])]

Pop D:
- add F
Queue: [(E,[A,C,E]), (F,[A,B,D,F])]

Pop F:
- goal reached → return [A,B,D,F]

Time: O(V+E) in typical BFS • Space: O(V).

15) Debugging Practice

Case 1: infinite loop

bug
forgot visited update
# if you never add to visited,
# BFS may keep revisiting nodes in cycles

Fix: add visited.add(node) when enqueuing.

Case 2: wrong neighbors

bug
bounds check wrong
# (r-1,c) might go out of bounds
# and crash or move into walls

Fix: validate bounds and allow only open cells.

Case 3: path not reconstructed

bug
missing parent pointers
# you return goal but cannot show route

Fix: store parent or store full path in the queue.

16) Common Errors

  • KeyError: missing node in adjacency list.
  • IndexError: grid out-of-bounds.
  • TypeError: neighbors not iterable.
  • Infinite loop: no visited tracking in cyclic graphs.

17) Best Practices

  • Always use a visited set for graph search.
  • Validate input graph/grid shapes.
  • Start with tiny examples and dry run by hand.
  • Document what queue/stack items represent.
  • Return both path and diagnostic info (concept).
