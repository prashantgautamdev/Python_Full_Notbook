1) Introduction

Humans repeat tasks: renaming files, copying folders, updating lists. Automation turns these steps into programs.

In this chapter, we build automation scripts step-by-step. We focus on safe file operations and clean structure.

2) Theory

2.1 Automation pattern

A typical automation workflow:

  • Read inputs (files, user arguments, online data).
  • Process (transform, filter, compute).
  • Write outputs (new files, reports, logs).
  • Verify (print results, checks).

2.2 Safety first

File automation can delete data by accident. Always test with a small folder first.

⚠ Warning

Avoid overwriting files without backup. Prefer “output to a new folder” approach.

2.3 Repeatability

Good automation scripts can run again and again and give the same result. Use functions and clear input/output.

3) Syntax

Python
basic file read/write
# Read a text file
with open("input.txt", "r", encoding="utf-8") as f:
    text = f.read()

# Write results to a new file
with open("output.txt", "w", encoding="utf-8") as f:
    f.write(text.upper())

print("Done. Created output.txt")
Python
processing loop (safe)
names = ["ravi", "asha", "neha"]

cleaned = []
for n in names:
    # step 1: strip spaces
    n2 = n.strip()
    # step 2: transform
    cleaned.append(n2.title())

print(cleaned)

Time: depends on file size or list length • Space: depends on output size.

Alternative: if you automate large files, stream line-by-line to reduce memory (later concepts).

4) Examples

4.1 Easy Example — rename-like report (no rename)

Instead of renaming files (dangerous), create a report that shows what you would do.

Python
generate report
import os

folder = "my_files"
for fname in os.listdir(folder):
    old_path = os.path.join(folder, fname)

    # Example rule: uppercase name without extension
    base, ext = os.path.splitext(fname)
    new_fname = base.upper() + ext

    new_path = os.path.join(folder, new_fname)

    print("Would rename:", old_path)
    print("To:", new_path)
    print("---")

Output: prints “Would rename …” lines.

4.2 Medium Example — extract emails from a text file (concept)

Python
simple extraction idea
import re

with open("message.txt", "r", encoding="utf-8") as f:
    text = f.read()

# Very basic email-like pattern
emails = re.findall(r"[\w\.-]+@[\w\.-]+", text)

# Remove duplicates
unique = sorted(set(emails))

with open("emails.txt", "w", encoding="utf-8") as f:
    for e in unique:
        f.write(e + "\n")

print("Saved", len(unique), "emails")

Output: creates emails.txt.

4.3 Advanced Example — build a reusable automation function

Python
function + dry run
import shutil
import os


def copy_filtered_files(src_folder, dst_folder, ext, dry_run=True):
    """Copy files ending with ext from src to dst.

    Time complexity: O(n) files in folder.
    Space complexity: O(1) extra (besides loop variables).
    """
    os.makedirs(dst_folder, exist_ok=True)

    for name in os.listdir(src_folder):
        if not name.endswith(ext):
            continue

        src_path = os.path.join(src_folder, name)
        dst_path = os.path.join(dst_folder, name)

        if dry_run:
            print("[dry-run] Would copy:", src_path, "→", dst_path)
        else:
            shutil.copy2(src_path, dst_path)
            print("Copied:", src_path)


# Try with dry_run first
copy_filtered_files("input_dir", "output_dir", ".txt", dry_run=True)

Output: dry-run prints what would be copied (no changes).

Time: O(n) • Space: O(1) extra.

    f.write(...)

bug
filter wrong

    ...

16) Common Errors

  • FileNotFoundError: wrong path or missing file.
  • PermissionError: cannot read/write that folder.
  • UnicodeDecodeError: wrong encoding used for reading.
  • IsADirectoryError: treated a folder as a file.

17) Best Practices

  • Start with dry_run and preview planned actions.
  • Write to a new folder instead of overwriting.
  • Use helper functions to keep logic clean.
  • Validate inputs (folder exists, extension format).
  • Use clear logging and friendly error messages.

22) Answer Key (Explain every answer)

Key explanations

  • Automation: using code to perform repeat steps for you.
  • dry_run: simulate actions and preview results without changing files.
  • Read file: open in read mode and call f.read().
  • Write file: open in write mode and call f.write().
  • os.listdir: lists names inside a folder so you can loop.
  • endswith: checks file extension so you filter correctly.
  • New folder: prevents accidental overwrites and makes rollback easy.
  • Verify: check counts, sample outputs, and output folder contents.
