1) Introduction

In Chapter 01, you learned that programs store values in variables. Now we go one step deeper: what are the common types of values in Python?

⭐ Remember

Variables hold values. Types tell Python how to treat those values. This prevents mistakes like adding text like a number.

2) Theory

📚 Big Idea: Types are rules

Type is not just a label. It also controls what operations are allowed. For example, numbers can be added. Strings can be joined.

2.1 Common data types (beginner set)

  • int: whole numbers like 3, -10
  • float: decimals like 3.14, -0.5
  • str: text like "hello"
  • bool: True or False

2.2 Python is dynamically typed

“Dynamically typed” means you do not declare types. Python figures them out at runtime. Example: x = 5 makes x an int. If later you do x = "5", it becomes a str.

⚠ Warning: Mixing types

Some combinations work, others fail. For example: "5" + 2 fails because Python cannot add text and number.

2.3 Type conversion (casting)

Conversion is turning a value into a different type. Common conversions:

  • int("10") → 10
  • float("3.5") → 3.5
  • str(10) → "10"

Conversion can also fail. Example: int("abc") raises an error.

💡 Output prediction

Consider this snippet:

Python
prediction

print(a + str(b))

Output should be 102 because 2 is converted to text.

Time complexity: O(1) (basic assignments) • Space: O(1) per value.

Alternative approach: use conversions only at the boundary (e.g., when reading input).

4) Examples

4.1 Easy Example — int, float, str

Python
types
count = 7          # int
price = 19.99      # float
label = "Item"     # str

print(count, type(count))
print(price, type(price))
print(label, type(label))

Output: Values with corresponding types like .

Time: O(1) • Space: O(1)

Alternative approach: infer type using isinstance(value, int) (later, but mentioned).

4.2 Medium Example — converting types safely

Python
casting
text_number = "42"
number = int(text_number)

pi_text = "3.14"
pi = float(pi_text)

print(number + 8)
print(pi * 2)

Output: 50 and 6.28

Time: O(n) for parsing strings of length n • Space: O(1) extra.

Alternative approach: validate input before converting.

4.3 Advanced Example — bool behaves like int (careful!)

Python
bool math
is_ok = True
is_fail = False

print(is_ok + 10)   # True behaves like 1
print(is_fail + 10) # False behaves like 0

Output: 11 then 10

Time: O(1) • Space: O(1)

⚠ Warning

This is true in Python, but you should not rely on it for readability. Prefer explicit logic comparisons.

Alternative approach: use boolean logic: if is_ok:.

8) Built-in Functions

Most useful in this chapter

  • type(x): what type is x?
  • int(x): convert to integer
  • float(x): convert to float
  • str(x): convert to string
  • len(s): number of characters (for strings)
Python
casting and length
x = 10
print(type(x))

s = str(x)
print(type(s))
print("length:", len(s))

Output: , then , then length 2.

Time: O(n) for len on string length n • Space: O(n) for storing string.

Alternative approach: avoid converting if you only need printing (use print directly).

9) Methods

Methods belong to objects. A string is an object, so it has string methods.

Python
string methods
word = "python"
print(word.upper())
print(word.capitalize())

Output: PYTHON then Python

Time: O(n) • Space: O(n) for new strings.

Alternative approach: use slicing and manual operations (later advanced).

16) Common Errors

  • TypeError: wrong types used with an operator (e.g., "5" + 2).
  • ValueError: conversion fails (e.g., int("abc")).
  • NameError: variable used before assignment (review from Chapter 01).

17) Best Practices

  • Convert explicitly when mixing types.
  • Keep conversions near where data enters the program (input boundary).
  • Use comments to document why you converted types.
  • Use descriptive variable names.
  • Use type() when confused.

18) Summary

You learned that variables store values and types describe how Python treats those values. Python decides types dynamically. You also learned conversion rules and why mixing types can raise errors.
