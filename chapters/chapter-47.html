1) Introduction

Interview questions are not random. They test how you think.

This chapter helps you prepare with a repeatable approach and practice drills.

2) Theory

2.1 The interview workflow

  1. Understand the problem and restate it.
  2. Plan with a data structure and approach.
  3. Code with clear variable names.
  4. Test using examples + edge cases.
  5. Analyze time and space complexity.

2.2 Complexity explanation

Many candidates can code. Fewer can explain complexity. You will learn to say: “I loop over n items once, so O(n).”

2.3 Debug mindset

  • Check off-by-one errors.
  • Validate assumptions about input types.
  • Print small intermediate values when stuck.

⚠ Warning

Don’t optimize prematurely. First make it correct, then make it faster.

3) Syntax

Python
example pattern: two-sum using dict
def two_sum(nums, target):
    """Return indices of two numbers that sum to target.

    Time complexity: O(n) average.
    Space complexity: O(n).
    Alternative: brute force O(n^2).
    """
    # value -> index mapping for fast lookup
    seen = {}

    for i, x in enumerate(nums):
        need = target - x

        # If we have seen the needed value, return indices
        if need in seen:
            return [seen[need], i]

        # Store current value
        seen[x] = i

    # If no solution, return None
    return None

print(two_sum([2,7,11,15], 9))  # Output: [0,1]

Output: indices list or None.

Time: O(n) • Space: O(n).

Alternative: nested loops (O(n²)).

4) Examples

4.1 Easy Example — reverse a string

Python
string manipulation
def reverse_words(s):
    """Reverse each word in a sentence.

    Time: O(n) • Space: O(n)
    Alternative: manual loop building string.
    """
    parts = s.split()
    reversed_parts = [p[::-1] for p in parts]
    return " ".join(reversed_parts)

print(reverse_words("hello world"))  # Output: "olleh dlrow"

4.2 Medium Example — frequency count

Python
count + top element
from collections import Counter

def most_frequent(nums):
    """Return the most frequent number.

    Time: O(n) average for counting • Space: O(k)
    Alternative: dict counts manually.
    """
    c = Counter(nums)
    return c.most_common(1)[0][0]

print(most_frequent([1,2,2,3,3,3]))  # Output: 3

Flow idea

15) Debugging Practice

Case 1: index off-by-one

bug
wrong loop range
for i in range(len(nums)-1):
    # misses last element

Fix: use correct boundaries like range(len(nums)).

Case 2: missing return for no solution

bug
None handling
# forgot return None

Fix: return None explicitly.

Case 3: wrong update order

bug
two-sum store timing
# storing x before checking need

Fix: check need first, then store current value (avoid reusing same element).

16) Common Errors

  • Time limit: algorithm too slow (wrong Big-O).
  • Wrong answer: missed edge case or off-by-one.
  • Stack overflow: recursion without base case.
  • KeyError: using dict without checking existence.

17) Best Practices

  • Speak your plan out loud.
  • Write tests mentally and with assert.
  • Always handle “no solution” cases.
  • Explain complexity clearly.
  • Use clear names and comments.
