🎯 Learning Objectives

By the end of this chapter, you will be able to:

  • Install Python and verify it works.
  • Choose an editor and run a Python file.
  • Create and activate a virtual environment (venv).
  • Understand pip and how dependencies are installed.

1) Introduction

Before coding, your computer must have Python installed correctly. Setup is like preparing your kitchen before cooking.

💡 Real-life example

If you install Python and libraries in the wrong way, one project can break another. Virtual environments solve that by separating dependencies.

2) Theory

📚 What is an “environment”?

An environment is the place where Python runs and where your packages (libraries) live. You can have different environments for different projects.

2.1 Python installation checklist

  • Confirm Python version with python --version
  • Confirm pip with pip --version
  • Ensure you can run python from the terminal

2.2 Editors (IDEs) vs simple text editors

An editor is where you write code. IDEs add helpful features like running code, autocompletion, and debugging.

Two common choices: VS Code and PyCharm. Either is fine.

2.3 Virtual environments (venv)

Virtual environments create an isolated Python + pip setup inside a folder. When you install packages, they go into that folder.

⚠ Warning

Do not install packages globally while learning. It makes dependency conflicts more likely.

3) Syntax

Most of the setup is command-line commands. Think of them as “terminal instructions” rather than Python code.

Terminal
Basic commands
# Check Python
python --version

# Start Python interactive mode
python

# Exit interactive mode
exit()

💡 Output prediction

If Python is installed, python --version prints something like: Python 3.x.x.

Time complexity: O(1) (just runs version checks) • Space: O(1).

Alternative approach: use py --version on Windows if needed.

4) Examples

4.1 Easy Example — create and run a script

Python
run a file
# file: hello.py
print("Hello from Chapter 02")

Output: Hello from Chapter 02

Time: O(1) • Space: O(1)

Alternative approach: run in interactive mode and type the print line.

4.2 Medium Example — virtual environment

Terminal
venv steps
# inside your project folder
python -m venv venv

# Windows activation
venv\Scripts\activate

# upgrade pip (recommended)
pip install --upgrade pip

After activation, your terminal prompt usually shows (venv).

Time: O(1) conceptual setup • Space: O(k) for installed packages (k depends on your installs).

Alternative approach: use Conda environments if you already use Anaconda.

4.3 Advanced Example — install dependencies

Terminal
pip install
# install requests in the active venv
pip install requests

# confirm installed package
python -c "import requests; print(requests.__version__)"

Output: A version number like 2.x.x

Time: O(k) depending on package download/installation size • Space: O(k).

Alternative approach: use a requirements.txt to install fixed versions.

14) Mini Project

🧪 Mini Project: Environment Check Reporter

Create a script that prints:

  • Python version
  • Whether running inside venv
  • Executable path
Python
example solution
# file: env_report.py
import sys

print("=== Environment Report ===")
print("Python version:", sys.version)
print("Executable path:", sys.executable)

# venv detection (simple heuristic)
in_venv = (hasattr(sys, 'real_prefix') or sys.prefix != getattr(sys, 'base_prefix', sys.prefix))
print("Running in venv:", in_venv)

Expected output: A report showing version and path. Running in venv prints True.

15) Debugging Practice

Case 1: “python is not recognized”

Fix by installing Python and ensuring PATH is set. Restart terminal.

Case 2: “ModuleNotFoundError”

Fix by activating the correct venv and installing the missing package with pip.

Case 3: Pip installs but import still fails

Likely installed into a different environment. Verify pip belongs to your active Python.

16) Common Errors

  • Command not found: Python not installed or PATH not set.
  • ModuleNotFoundError: dependency not installed in current env.
  • Version mismatch: package requires another Python version.
  • Permission error: lacking rights to install globally.

17) Best Practices

  • Use venv for every project.
  • Keep dependencies in requirements.txt.
  • Upgrade pip carefully inside venv.
  • Never commit the venv/ folder.
  • Read error messages and follow the line number and traceback.
