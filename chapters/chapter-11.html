1) Introduction

Earlier you learned lists, which are flexible. Sometimes you want a collection that you don’t change.

That’s where tuples help.

2) Theory

2.1 What is a tuple?

A tuple is an ordered collection like a list. The difference: tuples are immutable.

2.2 Immutability = cannot change items

Immutable means you cannot update an item like t[0] = .... But you can still read and slice.

2.3 Tuples vs lists (comparison table)

Feature List Tuple
Mutable? Yes No
Created with [ ] ( )
Typical use Data you update Fixed data
Safety Can change by accident More protected from mistakes

⚠ Warning

If a tuple contains a mutable object (like a list inside a tuple), that inner object can still change. This chapter focuses on immutability of the tuple container itself.

3) Syntax

Python
create + access
t = (10, 20, 30)
print(t[0])
print(t[-1])
Python
slicing
t = (1, 2, 3, 4, 5)
print(t[1:4]) # (2,3,4)
print(t[:2])  # (1,2)
print(t[3:])  # (4,5)

Time complexity: indexing O(1), slicing O(k) • Space: slicing new tuple/list depends; concept O(k).

Alternative approach: if you need updates, use list.

4) Examples

4.1 Easy Example — coordinates

Python
fixed point
point = (5, 7)

x = point[0]
y = point[1]

print("x:", x)
print("y:", y)

Output: x: 5 and y: 7

4.2 Medium Example — tuple vs list update

Python
immutability
t = (1, 2, 3)
# t[0] = 99  # ❌ error: tuples are immutable

print(t)

4.3 Advanced Example — tuple unpacking

Python
unpack
name_age = ("Ravi", 20)
name, age = name_age
print(name)
print(age)

Output: Ravi then 20

Time: O(1) • Space: O(1) extra (ignoring strings).

Alternative: access by index (less readable).

7) Dry Run

Python
unpacking dry run
pair = ("A", "B")
a, b = pair
print(a)
print(b)
Step What happens Values
1 Create tuple pair ("A","B")
2 Unpack into a and b a="A", b="B"
3 Print variables A then B

Time: O(1) • Space: O(1).

Alternative: read with pair[0] and pair[1].

8) Built-in Functions

  • len(t): number of items in tuple.
Python
len()
t = (10, 20, 30)
print(len(t))

Output: 3

Time: O(1) • Space: O(1).

Alternative: manual counting (not recommended).

9) Methods

Tuples have fewer methods than lists because tuples are immutable. But you can still use methods like count() and index().

Python
count + index
t = (1, 2, 2, 3)

print(t.count(2))
print(t.index(3))

Output: 2 then 3

Time: O(n) • Space: O(1).

Alternative: loop and count manually.

15) Debugging Practice

Case 1: trying to modify tuple

Python
bug
t = (1,2,3)
t[0] = 9

Fix: create a new tuple, or use a list if you need changes.

Case 2: missing comma for single element

Python
bug
t = (5)
print(type(t))

Issue: (5) is just an integer in Python.

Fix: t = (5,)

Case 3: wrong unpack size

Python
bug
t = (1,2,3)
a,b = t

Fix: match number of variables: a,b,c = t

16) Common Errors

  • TypeError: trying to change a tuple element.
  • ValueError: wrong unpack count.
  • IndexError: index out of range.
  • SyntaxError: wrong comma/brackets in tuple creation.

17) Best Practices

  • Use tuples for fixed data.
  • Use lists when you need to update items.
  • Use unpacking to improve readability when sizes match.
  • When you must “change” a tuple, create a new one.
  • Prefer t.count() and t.index() for simple queries.

18) Summary

Tuples are ordered collections like lists, but they are immutable. You can read items and slice, but you cannot modify tuple elements. Tuples are useful for fixed data such as coordinates or pairs. You also learned tuple unpacking and tuple methods count() and index().

Key explanations
