1) Introduction

When your programs grow, one single file becomes hard to manage. Modules and packages let you split code into smaller, reusable parts.

⭐ Remember

Module = one Python file (.py)
Package = a folder containing multiple modules

2) Theory

2.1 What is a module?

A module is a file like math_utils.py that contains functions/variables. When you import it, Python loads it and gives you access to its names.

2.2 Import basics

When you write import math_utils, you are importing the whole module namespace.

2.3 Import specific names

With from math_utils import add, you import only the name add. This can make code shorter.

2.4 What is a package?

A package is a folder (for example my_app/) with Python modules inside. A common requirement is an __init__.py file to mark it as a package.

⚠ Warning: circular imports

If module A imports B and module B imports A, Python can get stuck (circular import). Later you will learn strategies to break cycles.

3) Syntax

Python
import a module
# In file main.py
import math_utils

result = math_utils.add(2, 3)
print(result)
Python
import specific name
# In file main.py
from math_utils import add

print(add(10, 20))
Python
packages
# Folder structure:
# my_pkg/
#   __init__.py
#   calc.py

from my_pkg.calc import multiply
print(multiply(3, 4))

Time complexity: importing loads code once (startup cost) • Space: depends on loaded objects.

Alternative approach: copy/paste code into one file (not recommended for maintainability).

4) Examples

4.1 Easy Example — create a module and import it

Files you will create

math_utils.py and main.py in the same folder.

Python
math_utils.py
# math_utils.py

def add(a, b):
    # Adds two numbers and returns the result.
    return a + b

def square(x):
    # Returns x squared.
    return x * x
Python
main.py
# main.py
import math_utils

print(math_utils.add(2, 5))
print(math_utils.square(4))

Output: 7 then 16

Time: O(1) per function call • Space: O(1) extra.

Alternative: from math_utils import add, square.

4.2 Medium Example — from import

Python
from import
# main.py
from math_utils import add

print(add(100, 50))

Output: 150

Time: O(1) • Space: O(1).

4.3 Advanced Example — packages

Python
package calc.py
# my_pkg/calc.py

def multiply(a, b):
    return a * b
Python
main using package
# main.py
from my_pkg.calc import multiply

print(multiply(6, 7))

Output: 42

Time: O(1) • Space: O(1) extra.

15) Debugging Practice

Case 1: Module not found

Python
bug
# main.py
import maths_utils

Fix: make sure the filename matches exactly (math_utils.py), and run from the correct folder.

Case 2: Import name not found

Python
bug
# main.py
from math_utils import addition

Fix: check function name. If your function is add, import add not addition.

Case 3: circular import

Python
bug
# a.py
import b

# b.py
import a

Fix: move shared logic into a new module (for example shared.py) and import that.

16) Common Errors

  • ModuleNotFoundError: wrong filename, wrong folder, or wrong import path.
  • ImportError: module exists but requested name doesn’t exist.
  • NameError: using a name without importing it.
  • AttributeError: imported module but function name wrong.
  • Import loop (circular): modules depend on each other in a cycle.

17) Best Practices

  • Keep modules small and focused (single responsibility).
  • Use explicit imports instead of import *.
  • Avoid circular dependencies; extract shared utilities into their own module.
  • Protect “test code” using if __name__ == "__main__".
  • Use clear names for modules: string_tools.py, file_helpers.py.

18) Summary

Modules let you split code into separate files and reuse functions. You reuse them with import or from ... import .... Packages organize modules into folders and make large projects manageable. You also learned about common import mistakes and the idea of circular imports.

22) Answer Key (Explain every answer)

Key explanations (short)

  • Module: a single .py file containing code.
  • import: loads the module and provides a namespace.
  • from ... import: imports specific names into your file’s namespace.
  • Package: a folder grouping modules; typically contains __init__.py.
  • Circular import: cycle that can leave names undefined while modules load.
