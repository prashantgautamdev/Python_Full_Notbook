1) Introduction

In real life, decisions matter: if it is raining, take an umbrella. else, walk outside.

Conditional statements let your Python program do the same.

⭐ Remember

Conditions must be True or False. Comparisons like x > 5 produce booleans.

2) Theory

2.1 What is a condition?

A condition is an expression that evaluates to True or False. Examples:

  • age >= 18
  • password == "1234"
  • score > 60 and score < 90

2.2 The if / elif / else rule

  • if: first check
  • elif: extra checks (0 or more)
  • else: runs only if none of the previous checks are True

⚠ Warning: indentation defines blocks

Python uses indentation to know what belongs inside if, elif, and else. Incorrect indentation will break your program.

3) Syntax

Python
if / else structure
# Basic if / else
age = 17

if age >= 18:
    print("Adult")
else:
    print("Minor")
Python
if / elif / else
# Multiple conditions
score = 72

if score >= 90:
    grade = "A"
elif score >= 80:
    grade = "B"
elif score >= 70:
    grade = "C"
else:
    grade = "D"

print("Grade:", grade)

Time complexity: O(1) per condition check • Space: O(1).

Alternative approach: use a lookup table (advanced) when conditions are simple categories.

4) Examples

4.1 Easy Example — even or odd

Python
% and if
n = 9

if n % 2 == 0:
    print("Even")

    print("Odd")

Output: Odd

Time: O(1)

Alternative: use n & 1 (bitwise; later).

4.2 Medium Example — multiple checks

Python
age category
age = 14

if age < 13:
    category = "Kid"
elif age < 20:
    category = "Teen"

Output: Teen

4.3 Advanced Example — combine conditions

Python
eligibility
age = 22

if age >= 18 and has_id and (is_member or age >= 50):
    print("Eligible")
else:
    print("Not eligible")

Output: depends on values.

💡 Tips

Use parentheses in long conditions to make meaning obvious.

7) Dry Run

Python

score = 72

if score >= 90:
    print("A")

    print("B")

    print("C")
else:
    print("D")
Check Expression True/False Next

Time complexity: O(m) where m is number of checks (worst case) • Space: O(1).

Alternative: if ranges are fixed, use interval checks carefully (still same idea).

8) Built-in Functions

Conditional code often uses print. You may also check types with type() while debugging.

Python
debug type
x = 10

if type(x) == int:
    print("x is int")
else:
    print("x is not int")

Output: x is int

Time: O(1) • Space: O(1)

Alternative: use isinstance(x, int) (advanced note).

9) Methods

Methods can help generate booleans for conditions.

Python
string condition
email = input("Enter email: ")

if email.endswith("@gmail.com"):
    print("Gmail user")
else:
    print("Not Gmail")

Output: depends on input

Time: O(n) for checking string ending • Space: O(1) extra.

Alternative: use find() (concept; later).

🧪 Mini Project: Ticket Price Decision

Create a program that calculates ticket price based on:

  • age
  • is_student
  • day_type ("weekday" or "weekend")

Rules (example):

  • Base price: 500
  • Weekend +100

Python
solution idea
age_text = input("Enter age: ")

student_text = input("Are you a student? (yes/no): ")
is_student = (student_text.strip().lower() == "yes")

day_type = input("Day type (weekday/weekend): ").strip().lower()

if day_type == "weekend":
    price = price + 100

if is_student and age < 25:
    price = price - (price * 0.30)

16) Common Errors

  • SyntaxError: missing : after if/elif.
  • IndentationError: code inside if/elif/else is not indented correctly.
  • TypeError: comparing different types (e.g., string with int without conversion).
  • Logic error: conditions are written incorrectly (common with and/or).

17) Best Practices

  • Write simple conditions first, then combine them.
  • Use parentheses for clarity in complex conditions.
  • Keep the elif ladder in correct order (most specific first).
  • Convert input values to correct type before comparing.
  • Use small tests (try multiple inputs) to validate logic.

18) Summary

Conditional statements let Python choose what to do. Use if for the main check, elif for alternatives, and else as the fallback. Conditions are expressions that return booleans. Python block indentation controls what code runs.

19) Cheat Sheet

Part Syntax Meaning
if if cond: run block when cond is True
elif elif cond: run when previous checks were False
else else: run when all conditions are False
common operators and/or/not combine boolean checks

22) Answer Key (Explain every answer)

Quick explanations

  • if runs when cond is True because Python chooses the first matching branch.
  • elif is else-if used for additional checks after the first fails.
  • else runs last only when all earlier conditions are False.
  • Indentation defines which lines belong inside a block.
  • and/or/not combine boolean results: and needs both True, or needs at least one True, not flips.

Debugging recap

  • SyntaxError: missing : after conditions.
  • IndentationError: block lines are not indented.
  • Logic error: conditions use wrong operators or wrong order.
