1) Introduction

Many websites show information in HTML. Web scraping means reading that HTML and extracting data automatically.

This chapter uses a responsible beginner-friendly approach. You will focus on concepts and safe patterns.

2) Theory

2.1 Fetch → parse → extract

  • Fetch: download the HTML text from a URL.
  • Parse: turn HTML into a searchable structure.
  • Extract: select elements you want (titles, links, prices).

2.2 CSS selectors vs parsing methods (concept)

You can locate elements using tag names, attributes, and classes. In BeautifulSoup, you can find elements by find() / select() concepts.

2.3 Robots.txt and rate limiting

robots.txt tells you which paths may be scraped. Rate limiting avoids overloading servers.

⭐ Remember

Scraping is powerful. Do it politely: small request rate + correct content usage.

4) Examples

4.1 Easy Example — extract links

Python
extract all 
# concept example
import requests
from bs4 import BeautifulSoup

url = "https://example.com"
html = requests.get(url, timeout=10).text

soup = BeautifulSoup(html, "html.parser")
links = soup.find_all("a")

for a in links[:5]:
    href = a.get("href")
    text = a.get_text(strip=True)
    print(text, "→", href)

Output: prints the first few link texts and href targets (depends on page).

4.2 Medium Example — extract a table (concept)

Python
extract rows from table
# concept example
from bs4 import BeautifulSoup

soup = BeautifulSoup(html, "html.parser")

table = soup.find("table")
if not table:
    print("No table found")
else:
    rows = table.find_all("tr")
    for row in rows[:3]:
        cols = row.find_all("td")
        values = [c.get_text(strip=True) for c in cols]
        print(values)

4.3 Advanced Example — responsible scraping loop (concept)

Python
rate limiting idea
# concept example
import time
import requests
from bs4 import BeautifulSoup

urls = [
  "https://example.com/page1",
  "https://example.com/page2",
]

for i, url in enumerate(urls, start=1):
    print("Fetching:", url)
    html = requests.get(url, timeout=10).text
    soup = BeautifulSoup(html, "html.parser")

    # Do small extraction
    titles = [h.get_text(strip=True) for h in soup.find_all("h2")]
    print("Found titles:", titles[:3])

    # polite delay
    time.sleep(1.0)

15) Debugging Practice

Case 1: find returns None

bug
wrong tag or selector
h1 = soup.find("h1")
print(h1.text)  # crash if h1 is None

Fix: check if h1: before using h1.text.

Case 2: extraction returns empty list

bug
page structure changed
prices = soup.find_all(class_="price")
print(len(prices))  # 0

Fix: inspect HTML and update selectors.

Case 3: timeout or request error

bug
server slow or blocked
requests.get(url)

Fix: set timeout=, handle exceptions, and reduce request rate.

16) Common Errors

  • requests.exceptions.Timeout: the server didn’t respond in time.
  • requests.exceptions.ConnectionError: network or DNS issue.
  • AttributeError: you used .text on None.
  • Unicode issues: wrong decoding; use .text (requests).
  • Empty extraction: selectors don’t match current HTML.

17) Best Practices

  • Prefer official APIs.
  • Check robots.txt and follow rules.
  • Use timeouts and retries carefully.
  • Sleep between requests (rate limiting).
  • Use parsing libraries instead of regex for HTML.
  • Write extraction functions and test with saved HTML samples.
  • Handle missing elements safely.

22) Answer Key (Explain every answer)

Key explanations

  • Fetch: download HTML from the URL.
  • Parse: create a structure so you can search tags and attributes.
  • Extract: select elements and clean text/attributes.
  • find vs find_all: find returns first match, find_all returns a list of matches.
  • get_text(strip=True): returns readable text without extra whitespace at ends.
  • get("href"): returns the link target stored in the href attribute.
  • Timeout: prevents your program from waiting forever.
  • robots.txt + rate limiting: reduces legal/ethical risk and server load.
