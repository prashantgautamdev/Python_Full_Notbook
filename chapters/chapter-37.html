1) Introduction

Instead of writing rules like “if this then that”, machine learning finds patterns from data.

A model is trained on examples. After training, it predicts the outcome for new inputs.

2) Theory

2.1 Model, features, target

In ML we usually have:

  • Features: inputs you measure (x values).
  • Target: what you want to predict (y value).
  • Model: the function that maps features → prediction.

2.2 Supervised vs unsupervised

  • Supervised learning: target is known in training data.
  • Unsupervised learning: target is not given; model finds structure.

2.3 Training and evaluation (concept)

If you evaluate on the same data you trained on, results can look too good. That’s why we use separate test/validation data.

⚠ Warning

Overfitting means the model memorizes training data but fails on new data. Underfitting means the model is too simple.

3) Syntax

Concept
k-NN idea
# Pseudocode (concept)
# 1) store training points
# 2) for a new point:
#    - compute distance to all training points
#    - take k nearest labels
#    - predict most common label (classification)
Python
toy prediction with k-NN (no libraries)
import math
from collections import Counter


def knn_predict(train_points, train_labels, query, k=3):
    """
    train_points: list of (x, y)
    train_labels: list of labels (same length as train_points)
    query: (x, y)
    k: number of neighbors

    Time complexity: O(n) distances + O(n log n) if sorting.
    We’ll use sorting for simplicity.
    Space complexity: O(n) for distance list.
    """
    distances = []

    for point, label in zip(train_points, train_labels):
        # Euclidean distance
        dx = point[0] - query[0]
        dy = point[1] - query[1]
        dist = math.sqrt(dx * dx + dy * dy)
        distances.append((dist, label))

    # sort by distance (smallest first)
    distances.sort(key=lambda x: x[0])

    # take k closest labels
    k_labels = [label for _, label in distances[:k]]

    # choose most common label
    most_common = Counter(k_labels).most_common(1)[0][0]
    return most_common


train_points = [(0, 0), (1, 1), (2, 2), (1, 0)]
train_labels = ["cat", "cat", "dog", "cat"]

query = (1.2, 1.0)
pred = knn_predict(train_points, train_labels, query, k=3)

print("Prediction:", pred)

Output: “Prediction: …” depends on which label is most common among the k nearest points.

Time: roughly O(n log n) because we sort all distances • Space: O(n).

Alternative: use libraries like scikit-learn for real projects.

4) Examples

4.1 Easy Example — classification with majority vote

Explanation
what k controls
k = 1 → predict the closest point’s label
k = 3 → predict the majority label among 3 closest points
k too small: noisy predictions
k too large: may ignore local patterns

4.2 Medium Example — regression concept

Concept
predict average
For regression, neighbors’ y values are numeric.
Prediction can be the average of y for k nearest points.

4.3 Advanced Example — evaluation idea

Flow idea

Train → predict on test → compare predictions with true targets.

Concept
accuracy formula
accuracy = correct_predictions / total_predictions
