1) Introduction

When you open a website, your browser must talk to a server. That “talk” happens over a network.

In this chapter, we learn a low-level tool: Python’s sockets. You will build a tiny client and a tiny server and watch messages move.

2) Theory

2.1 Client vs Server (mental model)

A server waits for connections. A client connects and sends a request. The server sends a response.

2.2 IP address + port

Computers on a network are identified by an IP address. Inside a computer, each program can listen on a port.

For learning, we usually use 127.0.0.1 (meaning “my own computer”).

2.3 Bytes vs text

Sockets send bytes, not Python strings. So if you send text, you must convert:

  • encode: string → bytes (usually UTF-8)
  • decode: bytes → string (UTF-8)

⚠ Warning

Message boundaries are tricky. TCP is a byte stream. Your recv() call might return part of a message.

2.4 Message framing (delimiter or length prefix)

To know where one message ends and the next begins, you need a simple protocol rule. Two common ideas are:

  • Delimiter: end each message with a newline \n
  • Length prefix: first send the length, then send that many bytes

3) Syntax

Python
basic TCP server skeleton
import socket

HOST = "127.0.0.1"
PORT = 5001

# Create an IPv4 TCP socket
server_sock = socket.socket(socket.AF_INET, socket.SOCK_STREAM)

# Tell the OS: listen on this host/port
server_sock.bind((HOST, PORT))

# Allow up to 1 queued connection (learning example)
server_sock.listen(1)

print("Server waiting for a connection...")

# accept() returns a connected socket + the client address
conn, client_addr = server_sock.accept()
print("Connected to:", client_addr)

# Receive some bytes (up to 1024 bytes)
data = conn.recv(1024)
print("Received bytes:", data)

# Reply (must be bytes)
conn.sendall(b"Hello from server")

# Always close sockets
conn.close()
server_sock.close()
Python
basic TCP client skeleton
import socket

HOST = "127.0.0.1"
PORT = 5001

client_sock = socket.socket(socket.AF_INET, socket.SOCK_STREAM)

# connect to server (client initiates)
client_sock.connect((HOST, PORT))

# send bytes
client_sock.sendall(b"Hello server")

# receive bytes reply
reply = client_sock.recv(1024)
print("Reply bytes:", reply)

client_sock.close()

Time: network/IO bound • Space: proportional to message size.

Alternative: use requests for HTTP APIs (higher-level).

4) Examples

4.1 Easy Example — Echo server (send back same bytes)

Python
echo server
import socket

HOST = "127.0.0.1"
PORT = 5002

server = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
server.bind((HOST, PORT))
server.listen(1)

conn, addr = server.accept()
print("Connected:", addr)

data = conn.recv(1024)
print("Got:", data)

# Echo back the same bytes
conn.sendall(data)

conn.close()
server.close()

Output: client receives exactly the same bytes it sent.

4.2 Medium Example — text encode/decode

Python
server decodes text and replies
import socket

HOST = "127.0.0.1"
PORT = 5003

server = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
server.bind((HOST, PORT))
server.listen(1)

conn, _ = server.accept()
raw = conn.recv(1024)

# Convert bytes → string
text = raw.decode("utf-8")

reply_text = f"Server got: {text}"
reply_bytes = reply_text.encode("utf-8")

conn.sendall(reply_bytes)

conn.close()
server.close()

Output: string reply like Server got: ....

4.3 Advanced Example — framing idea with newline delimiter

Conceptually, if every message ends with a newline (\n), you can read until you see that delimiter.

Python
framing concept snippet
# Concept only (not complete server):
# 1) client sends: b"Hello\n"
# 2) server reads chunks and accumulates until it sees b"\n"
# 3) then you process exactly one complete message

15) Debugging Practice

Case 1: Connection refused

Python
symptom
# client attempts to connect but nothing listens
client_sock.connect(("127.0.0.1", 5001))
# ConnectionRefusedError

Fix: start the server first and ensure client/server ports match.

Case 2: UnicodeDecodeError

Python
bug
text = raw.decode("utf-8")

Fix: ensure sender encoded with UTF-8. For learning, use errors="replace".

Case 3: Server reads only part of message

Python
bug
data = conn.recv(1024)
# long message may arrive in multiple chunks

Fix: implement framing (delimiter or length prefix) and read until complete.

16) Common Errors

  • ConnectionRefusedError: server isn’t listening on that host/port.
  • TimeoutError: server didn’t respond within timeout.
  • UnicodeDecodeError: bytes aren’t valid UTF-8 for your decode.
  • BrokenPipeError: client disconnected while sending/receiving.
  • OSError: port already in use or wrong address family.

17) Best Practices

  • Use 127.0.0.1 for safe local learning.
  • Always encode/decode text around sockets.
  • Use a framing strategy (delimiter or length prefix) once messages can be multi-part.
  • Close sockets with try/finally or safe cleanup.
  • Handle exceptions and send friendly error messages where possible.

22) Answer Key (Explain every answer)

Key explanations

  • Client/server: clients connect and send data; servers accept connections and respond.
  • Bytes: sockets transfer raw bytes; text must be encoded/decoded.
  • Message framing: because TCP can split/merge bytes, you need a rule to detect message boundaries.
  • Connection refused: typically means nothing is listening on that host/port.
  • Encoding: UTF-8 ensures consistent text representation across machines.
