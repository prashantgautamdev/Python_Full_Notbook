1) Introduction

A web page is usually controlled by a browser. A desktop GUI is controlled by a Python program.

In this chapter, we use tkinter (built into many Python installs) to build a small desktop application.

2) Theory

2.1 Event-driven programming

In event-driven programming, you do not run a “big loop” yourself. Instead, you register handlers (functions) for events.

2.2 Widgets, layout, and the main loop

  • Widgets: UI elements like Label, Button, Entry.
  • Layout: where widgets appear (grid/pack/place).
  • Main loop: keeps the window running and dispatches events.

2.3 Callbacks (functions on click)

When the user clicks a button, tkinter calls the callback you provided.

⚠ Warning

Do not block the GUI thread with long tasks. Long work should be moved to a worker thread (advanced topic).

3) Syntax

Python
minimal tkinter window
import tkinter as tk

# 1) Create the main application window
root = tk.Tk()

# 2) Set window title (optional)
root.title("My First GUI")

# 3) Add widgets
label = tk.Label(root, text="Hello from tkinter!", font=("Arial", 14))
label.pack(padx=16, pady=16)

# 4) Start the GUI event loop
root.mainloop()

Time: GUI loop runs continuously until the window closes • Space: UI objects kept in memory.

Alternative: use other GUI frameworks (PyQt, Kivy) for advanced features.

4) Examples

4.1 Easy Example — Label + Button

Python
callback updates label
import tkinter as tk

root = tk.Tk()
root.title("Counter")

# State variable (count)
count = 0

label = tk.Label(root, text="Count: 0", font=("Arial", 16))
label.pack(padx=16, pady=10)


def increment():
    # Use nonlocal to modify count inside callback
    nonlocal count
    count += 1
    label.config(text=f"Count: {count}")

# Button calls increment when clicked
btn = tk.Button(root, text="Add 1", command=increment)
btn.pack(padx=16, pady=10)

root.mainloop()

Output: each click updates the label text.

4.2 Medium Example — Entry + Submit

Python
read user input
import tkinter as tk

root = tk.Tk()
root.title("Greeter")

info = tk.Label(root, text="Type your name and press Submit")
info.pack(padx=16, pady=8)

name_entry = tk.Entry(root, width=30)
name_entry.pack(padx=16, pady=8)

result = tk.Label(root, text="")
result.pack(padx=16, pady=8)


def submit():
    name = name_entry.get().strip()
    if not name:
        result.config(text="Please enter a name.")
    else:
        result.config(text=f"Hello, {name}!")

submit_btn = tk.Button(root, text="Submit", command=submit)
submit_btn.pack(padx=16, pady=12)

root.mainloop()

Output: shows greeting or an error message.

4.3 Advanced Example — Using grid layout with Frames

Python
frames + grid
import tkinter as tk

root = tk.Tk()
root.title("Two-Panel")

# Frame A
left = tk.Frame(root, borderwidth=1, relief="solid")
left.grid(row=0, column=0, padx=10, pady=10)

# Frame B
right = tk.Frame(root, borderwidth=1, relief="solid")
right.grid(row=0, column=1, padx=10, pady=10)

left_title = tk.Label(left, text="Left Panel", font=("Arial", 14))
left_title.grid(row=0, column=0, padx=12, pady=8)

right_title = tk.Label(right, text="Right Panel", font=("Arial", 14))
right_title.grid(row=0, column=0, padx=12, pady=8)

root.mainloop()

Output: two bordered panels side by side.

15) Debugging Practice

Case 1: Button does nothing

Bug
callback not connected
btn = tk.Button(root, text="Click")
# missing command=...

Fix: use tk.Button(..., command=my_callback).

Case 2: Window closes immediately

Bug
no mainloop
root = tk.Tk()
# forgot root.mainloop()

Fix: call root.mainloop() at the end.

Case 3: Layout overlaps

Bug
mix pack + grid
label.pack()
button.grid(row=0, column=0)

Fix: pick one layout method per container (pack or grid).

Case 4: Callback crashes with ValueError

Bug
invalid number parsing
n = int(entry.get())
# user typed "abc"

Fix: wrap parsing in try/except and show error in GUI.

16) Common Errors

  • AttributeError: widget variable not created.
  • TypeError: wrong callback signature (concept).
  • ValueError: parsing invalid Entry text to int/float.
  • Layout issues: mixing pack/grid in same container.
  • GUI freeze: long blocking work inside callbacks.

17) Best Practices

  • Keep GUI code small; move logic into helper functions.
  • Validate inputs from entries before computing.
  • Update widgets using config() rather than recreating them.
  • Avoid blocking operations inside callbacks.
  • Use consistent widget variable names (e.g., name_entry, submit_btn).

22) Answer Key (Explain every answer)

Key explanations

  • Event-driven: code runs when an event occurs (click/type).
  • mainloop: runs the event dispatcher and keeps window alive.
  • Button command: tells tkinter which function to call.
  • Entry.get(): reads current text the user typed.
  • pack vs grid: pack is simple stacking; grid positions by row/column.
  • Separate logic: easier testing and fewer UI bugs.
