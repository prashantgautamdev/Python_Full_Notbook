1) Introduction

Text data often contains patterns. Example: emails, phone numbers, dates, IDs.

Regex lets you search and extract those patterns using rules. Instead of writing many loops, you write a pattern.

2) Theory

2.1 Regex = pattern language

A regex pattern is like a mini-rule describing what you want. For example, \d means “a digit”.

2.2 Match vs Search

  • re.match: checks at the start of the string.
  • re.search: finds the pattern anywhere.

2.3 findall extracts all matches

re.findall returns a list of all matching parts.

⭐ Remember Box

Think: match = start, search = anywhere, findall = extract all.

3) Syntax

Python
import re + search
import re

text = "Order ID: 12345"

m = re.search(r"ID: (\\d+)", text)
if m:
    print("Found number:", m.group(1))
Python
findall digits
text = "A1 B20 C300"
nums = re.findall(r"\\d+", text)
print(nums)

Time: depends on pattern complexity and text length • Space: depends on matches you store.

Alternative: use plain string operations for simple tasks, but regex is powerful for patterns.

4) Examples

4.1 Easy Example — find digits

Python
digits extraction
import re

text = "Room 401, Floor 4"

digits = re.findall(r"\\d+", text)
print(digits)

Output: ['401', '4']

4.2 Medium Example — simple email pattern

Python
email-like extraction
import re

text = "Contact: ravi@example.com or asha@test.org"

emails = re.findall(r"[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\\.[A-Za-z]{2,}", text)
print(emails)

Output: two emails (order depends on text).

4.3 Advanced Example — group and validate parts

Python
group() for pieces
import re

text = "Date: 2026-07-13"
pattern = r"Date: (\\d{4})-(\\d{2})-(\\d{2})"

m = re.search(pattern, text)
if m:
    year = m.group(1)
    month = m.group(2)
    day = m.group(3)
    print("Y/M/D:", year, month, day)

Output: 2026 07 13

15) Debugging Practice

Case 1: forgetting raw string

Python
bug
import re
pattern = "\d+"
print(re.findall(pattern, "a1b2"))

Explanation: Python interprets \d inside a normal string. Use r"\d+" or escape properly.

Case 2: match returns None

Python
bug
import re
text = "hello Python"
print(re.match(r"Python", text))

Fix: use re.search or add ^ correctly.

Case 3: wrong group index

Python
bug
import re
text = "2026-07-13"
pattern = r"(\\d{4})-(\\d{2})-(\\d{2})"

m = re.search(pattern, text)
print(m.group(3))
print(m.group(1))

Explanation: group(1) is year, group(2) month, group(3) day. Confirm indexes.

16) Common Errors

  • TypeError/ValueError: if you pass non-string text (concept).
  • NoneType: calling group() when search returned None.
  • Regex syntax error: invalid pattern characters.
  • Wrong escaping: backslashes not handled correctly.

17) Best Practices

  • Start with small patterns; test with a short example string.
  • Use raw strings r"..." for regex patterns.
  • Prefer readable patterns; avoid extremely complex one-liners.
  • Always check if m: before using m.group().
  • Compile patterns for repeated use (advanced).

Regex is a powerful text pattern tool in Python. You learned re.search (find anywhere), re.match (start), and re.findall (extract all). You practiced with digits, groups, and practical extraction tasks. You also learned debugging and common mistakes with escaping and None results.
