1) Introduction

In programming, you often repeat tasks. Example: print numbers 1 to 5, or check several items in a list.

Real-life example

Imagine you want to count 10 steps. You do the same action again and again. That repeating action is what loops do in code.

2) Theory

2.1 for loop vs while loop

  • for: repeat over a sequence (like numbers from range()).
  • while: repeat as long as a condition remains True.

2.2 range()

range(n) generates numbers from 0 to n-1. You can also use range(start, end) and range(start, end, step).

2.3 Loop control: break and continue

  • break: exit the loop immediately.
  • continue: skip the current iteration and move to the next.

⚠ Warning: infinite loops

A while loop needs a condition that will eventually become False. Otherwise, the loop never stops.

3) Syntax

Python
for loop
for i in range(5):
    print(i)
Python
while loop
count = 0
while count < 3:
    print(count)
    count = count + 1
Python
break / continue
n = 0
while True:
    n = n + 1
    if n == 3:
        break
    if n % 2 == 0:
        continue
    print("n =", n)

Time complexity: O(number of iterations) • Space: O(1) extra (not counting output).

Alternative approach: use built-ins like sum() when the task is just aggregation.

4) Examples

4.1 Easy Example — print 1..5

Python
range + for
for x in range(1, 6):
    print(x)

Output: 1 2 3 4 5 (each on new line)

4.2 Medium Example — sum numbers

Python
accumulator
total = 0
for i in range(1, 6):
    total = total + i

print("Total:", total)

Output: Total: 15

Time: O(n) • Space: O(1)

Alternative: sum(range(1,6)).

4.3 Advanced Example — stop early with break

Python
search in a loop
numbers = [2, 5, 7, 9]
found = False

for n in numbers:
    if n == 7:
        found = True
        break

print("Found 7?", found)

Output: True

Time: O(n) worst case • Space: O(1) extra.

6) Memory Visualization

Each loop iteration updates the loop variable (like i). Values are processed one-by-one.

Iteration i value What happens
1 1 Add 1 to total
2 2 Add 2 to total
3 3 Add 3 to total

🧠 Memory trick

for = “for each value in the list/sequence”. while = “while this condition is True, keep repeating”.

8) Built-in Functions

  • range(): create loop numbers.
  • sum(): add numbers (alternative to manual loops).
  • min()/max(): find extremes.
Python
sum() alternative
print(sum(range(1, 6)))

Output: 15

9) Methods

Many loop tasks involve strings/lists. String methods may be used inside loops (example: check length).

Python
length check
words = ["a", "hello", "py"]
for w in words:
    if len(w) > 2:
        print(w)

Output: hello

Time: O(total characters processed)

Alternative: use list comprehension later.

15) Debugging Practice

Case 1: missing indentation

Python
bug
for i in range(3):
print(i)

Fix: indent print(i) inside the loop.

Case 2: wrong range boundaries

Python
bug
for i in range(1, 5):
    print(i)

Issue: prints 1..4 not 1..5.
Fix: use range(1, 6).

Case 3: infinite while

Python
bug
count = 0
while count < 3:
    print(count)

Fix: update the counter inside loop: count += 1.

16) Common Errors

  • IndentationError: loop body not indented.
  • Infinite loop: condition never becomes False.
  • Off-by-one: wrong range end value.
  • IndexError: list index out of range in loops.

17) Best Practices

  • Start with small loops and test quickly.
  • Use range(start,end) correctly (end excluded).
  • For while loops, ensure the condition changes every iteration.
  • Prefer total += i for readability.
  • Use break to avoid extra work when you already found the answer.

18) Summary

Loops repeat tasks. Use for loops with range() when you know the count. Use while loops when you repeat until a condition stops. Use break and continue for loop control.

19) Cheat Sheet

Need Code Meaning
Repeat N times for i in range(N): i goes 0..N-1
Repeat from start range(a,b) a..b-1
Repeat until condition while cond: run while True
Exit early break stop loop
Skip iteration continue next iteration

22) Answer Key (Explain every answer)

Key explanations

  • for iterates over a sequence produced by range().
  • while keeps running while a condition is True.
  • break exits immediately to code after the loop.
  • continue skips the remaining body and starts the next iteration.
  • Off-by-one happens because range end value is excluded.
