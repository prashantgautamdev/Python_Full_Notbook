1) Introduction

When you build software, you repeat similar design tasks. Examples: creating objects, switching algorithms, notifying many objects.

Design patterns capture proven solutions so you can reuse them.

2) Theory

2.1 Three pattern categories

  • Creational: how objects are created.
  • Structural: how classes/objects are composed.
  • Behavioral: how objects communicate and behave.

2.2 Common patterns (what they solve)

  • Singleton: one instance globally (careful usage).
  • Factory: create objects without calling concrete classes directly.
  • Strategy: swap algorithms at runtime.
  • Observer: notify subscribers when something changes.
  • Adapter: make one interface work with another.
  • Decorator: add features without changing core code.

⚠ Warning

Patterns are tools, not rules. If the problem is small, overusing patterns makes code harder to read.

3) Syntax

Python
Strategy pattern (interface + swapping)
from abc import ABC, abstractmethod

class PaymentStrategy(ABC):
    @abstractmethod
    def pay(self, amount):
        pass

class CreditCard(PaymentStrategy):
    def pay(self, amount):
        return f"Paid {amount} using credit card"

class Upi(PaymentStrategy):
    def pay(self, amount):
        return f"Paid {amount} using UPI"

class Checkout:
    def __init__(self, strategy: PaymentStrategy):
        self.strategy = strategy

    def checkout(self, amount):
        # Behavior changes depending on strategy
        return self.strategy.pay(amount)

cc = CreditCard()
upi = Upi()

print(Checkout(cc).checkout(200))
print(Checkout(upi).checkout(200))

Time: O(1) for selecting strategy + depends on pay • Space: O(1).

Alternative: use functions as strategies (simpler in Python).

4) Examples

4.1 Easy Example — Factory pattern (create by type)

Python
factory without direct class usage
class Dog:
    def speak(self):
        return "Bark"

class Cat:
    def speak(self):
        return "Meow"

def animal_factory(kind: str):
    # Time: O(1) selection
    if kind == "dog":
        return Dog()
    if kind == "cat":
        return Cat()
    raise ValueError("Unknown kind")

animal = animal_factory("cat")
print(animal.speak())  # Output: Meow

4.2 Medium Example — Decorator pattern

Python
add logging feature
def log_calls(func):
    # Decorator adds behavior around func
    def wrapper(*args, **kwargs):
        print("Calling:", func.__name__)
        result = func(*args, **kwargs)
        print("Returned:", result)
        return result
    return wrapper

@log_calls
def add(a, b):
    # Example core logic
    return a + b

print(add(3, 4))
# Output prints logs and then 7

4.3 Advanced Example — Observer pattern (subscribe/notify)

Python
simple event system
class Observable:
    def __init__(self):
        self.subscribers = []

    def subscribe(self, callback):
        self.subscribers.append(callback)

    def notify(self, message):
        # Call all subscribers
        for cb in self.subscribers:
            cb(message)

obs = Observable()

def listener1(msg):
    print("Listener1 got:", msg)

def listener2(msg):
    print("Listener2 got:", msg)

obs.subscribe(listener1)
obs.subscribe(listener2)

obs.notify("Update!")

Output: both listeners print the message.

7) Dry Run

Dry run
Decorator call order
@log_calls
def add(a,b): return a+b

When calling add(3,4):
1) wrapper runs
2) wrapper prints "Calling: add"
3) wrapper calls original add(3,4)
4) wrapper prints "Returned: 7"
5) wrapper returns 7

Time: wrapper adds constant overhead per call.

15) Debugging Practice

Case 1: decorator forgets to return

bug
missing return
def wrapper(*args, **kwargs):
    func(*args, **kwargs)
    # forgot: return result

Fix: store result and return result.

Case 2: Observer modifies list while iterating

bug
subscriber list change
for cb in self.subscribers:
    self.subscribers.remove(cb)

Fix: iterate over a copy: for cb in self.subscribers[:].

Case 3: Strategy passed wrong type

bug
runtime mismatch
checkout = Checkout("not a strategy")

Fix: validate strategy implements required method (concept).

16) Common Errors

  • TypeError: passing wrong strategy or callback type.
  • AttributeError: calling method that does not exist on a swapped object.
  • Logic bug: factory returning wrong class for a given kind.
  • Unexpected behavior: decorator closure uses wrong variables.

17) Best Practices

  • Use patterns when there is a real repeated design need.
  • Prefer composition over inheritance in many cases.
  • Keep interfaces small (one main method) to reduce coupling.
  • Document why a pattern is chosen.
  • Write small tests for each pattern implementation.

22) Answer Key (Explain every answer)

Key explanations

  • Design pattern: a proven solution structure for recurring design problems.
  • Categories: creational, structural, behavioral.
  • Strategy: keeps core context same while swapping algorithm behavior.
  • Factory: chooses which concrete object to create based on inputs.
  • Decorator: wraps functions/objects to add extra behavior.
  • Observer: lets many subscribers respond when an event occurs.
