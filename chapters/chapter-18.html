1) Introduction

Earlier you used loops. Sometimes you want a “shorter and clearer” way to build a new list/dict/set from existing data.

That is where comprehensions and generators help.

2) Theory

2.1 Comprehension idea

A comprehension is a compact way to apply an expression to each item. It can also filter items with if.

2.2 Iterators and generators (concept)

  • Iterator: an object that produces values one by one.
  • Generator: an iterator made with yield.

⚠ Warning

Generators are lazy: they compute values only when you ask for them. Don’t expect them to be fully stored in memory.

3) Syntax

Python
list comprehension
nums = [1, 2, 3, 4]

squares = [n * n for n in nums]
print(squares)
Python
with condition
nums = [1, 2, 3, 4, 5]

evens = [n for n in nums if n % 2 == 0]
print(evens)
Python
generator using yield
def count_up_to(n):
    i = 1
    while i <= n:
        yield i
        i += 1

for x in count_up_to(3):
    print(x)

Time: depends on how many items you generate/process • Space: generator saves memory.

Alternative: use regular loops.

4.3 Advanced Example — generator filter

Python
lazy generation
def filtered_evens(nums):
    for n in nums:
        if n % 2 == 0:
            yield n

nums = [1,2,3,4,5,6]

for x in filtered_evens(nums):
    print(x)

Output: 2 then 4 then 6

7) Dry Run

Python
generator dry run
def count_up_to(n):
    i = 1
    while i <= n:
        yield i
        i += 1

g = count_up_to(3)
print(next(g))
print(next(g))
Call What happens Returned
next(g) runs until first yield 1
next(g) continues after yield 2

Time: O(k) for k yielded values • Space: small extra memory.

9) Methods

Generators don’t have “methods” for calculation. They are iterators. Their useful interface is __iter__ and __next__ (concept).

15) Debugging Practice

Case 1: wrong comprehension order

Python
bug
nums = [1,2,3]
# Wrong syntax (expression after for)
print([n for n*n in nums])

Fix: [n*n for n in nums]

Case 2: generator used like a list

Python
bug
gen = (n*n for n in range(3))
print(gen[0])

Fix: convert to list list(gen) or iterate.

Case 3: generator exhausted

Python
bug
gen = (n for n in range(3))
for x in gen:
    pass
for y in gen:
    print(y)  # nothing

Fix: create a new generator or convert to list.

16) Common Errors

  • SyntaxError: wrong comprehension placement.
  • TypeError: treating generator like list (indexing).
  • Logic error: wrong condition leading to missing values.

17) Best Practices

  • Use comprehensions for simple transformations.
  • If it becomes hard to read, use a normal loop.
  • Prefer generators for large or streaming data.
  • Don’t use complicated nesting early in learning.
