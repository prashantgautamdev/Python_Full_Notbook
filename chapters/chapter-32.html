1) Introduction

When your program fails, print() helps only if you added it earlier. Logging gives structured information and can be turned up/down by level.

In this chapter, you learn Python’s built-in logging module.

2) Theory

2.1 Levels

Logging levels tell how serious a message is:

  • DEBUG: detailed developer info
  • INFO: normal progress
  • WARNING: something might go wrong
  • ERROR: operation failed
  • CRITICAL: system is unusable

2.2 Handlers (concept)

A handler decides where logs go: console, file, or other systems.

⚠ Warning

Don’t log secrets like passwords or API keys. Mask sensitive values.

3) Syntax

Python
basic logging
import logging

logging.basicConfig(
    level=logging.INFO,  # show INFO and above
    format="%(levelname)s: %(message)s"
)

logging.debug("This debug is hidden")
logging.info("Program started")
logging.warning("Low memory warning")
logging.error("Something failed")

Output: typically shows INFO, WARNING, ERROR lines only.

Time: O(1) per log call • Space: small (message string).

Alternative: use print()

4) Examples

4.1 Easy Example — logging around a risky operation

Python
try/except + logs
import logging

logging.basicConfig(level=logging.INFO, format="%(levelname)s: %(message)s")

def parse_int(s):
    # Convert string to int
    try:
        value = int(s)
        logging.info("Parsed %r into %d", s, value)
        return value
    except ValueError:
        logging.error("Could not parse %r as int", s)
        return None

print(parse_int("12"))
print(parse_int("abc"))

Output: logs + prints 12 then None.

4.2 Medium Example — log to a file (concept)

Python
file handler idea
# Concept: file logs
import logging

logger = logging.getLogger("app")
logger.setLevel(logging.DEBUG)

# In real code you'd add a FileHandler to send messages to a file.
# For this chapter, focus on the idea: handlers decide destinations.

4.3 Advanced Example — conditional debug (concept)

Python
avoid expensive debug strings
import logging

logging.basicConfig(level=logging.WARNING)

for i in range(3):
    # Only build message if DEBUG is enabled (concept pattern)
    logging.debug("Index=%s", i)

logging.warning("Finished loop")

Output: only WARNING appears if level is WARNING.

15) Debugging Practice

Case 1: no logs appear

bug
wrong log level
logging.basicConfig(level=logging.ERROR)
logging.info("hello")

Fix: set level to INFO or lower.

Case 2: logs show duplicates

bug
handlers added twice (concept)
# Adding handlers multiple times can duplicate messages

Fix: configure logging once and avoid adding handlers repeatedly.

Case 3: you log secrets

bug
security risk
logging.info("password=%s", password)

Fix: never log secrets. Mask or remove sensitive fields.

16) Common Errors

  • No output: logging level too high.
  • Duplicate output: handlers configured more than once.
  • Formatting issues: wrong format placeholders (concept).
  • Crash: log code used invalid message objects (rare, but handle carefully).

17) Best Practices

  • Use the correct level for each message.
  • Include useful context: IDs, file names, counts.
  • Keep messages short but meaningful.
  • Never log secrets.
  • Prefer logger.info("x=%s", x) style parameter formatting.
