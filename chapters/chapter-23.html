1) Introduction

Programs need to remember data even after the program closes. A database is designed for this.

In this chapter, we use SQLite because it is built into Python. It is perfect for learning.

2) Theory

2.1 Database basics

A database contains tables. A table has rows (records) and columns (fields).

2.2 CRUD operations

  • Create → insert rows
  • Read → select rows
  • Update → modify rows
  • Delete → remove rows

2.3 SQL injection concept

If you build SQL queries by string concatenation, attackers can inject extra SQL. Parameterized queries prevent this.

⚠ Warning

Always use parameters (?) instead of directly inserting user input into SQL strings.

3) Syntax

Python
connect + cursor
import sqlite3

conn = sqlite3.connect("app.db")
cur = conn.cursor()

cur.execute("SELECT 1")
print(cur.fetchone())

conn.close()
SQL
create table
CREATE TABLE users(
  id INTEGER PRIMARY KEY,
  name TEXT NOT NULL,
  age INTEGER
);
Python
parameterized insert
cur.execute(
    "INSERT INTO users(name, age) VALUES(?, ?)",
    ("Ravi", 20)
)
conn.commit()

Time: depends on query • Space: depends on result size.

Alternative: use ORM libraries (advanced), but SQLite + SQL is best for learning.

4) Examples

4.1 Easy Example — create a table + insert

Python
setup db
import sqlite3

conn = sqlite3.connect("app.db")
cur = conn.cursor()

cur.execute("""
CREATE TABLE IF NOT EXISTS users(
  id INTEGER PRIMARY KEY,
  name TEXT NOT NULL,
  age INTEGER
)
""")

cur.execute(
  "INSERT INTO users(name, age) VALUES(?, ?)",
  ("Asha", 18)
)
conn.commit()

conn.close()
print("User inserted")

Output: User inserted

4.2 Medium Example — read/select rows

Python
select
import sqlite3

conn = sqlite3.connect("app.db")
cur = conn.cursor()

cur.execute("SELECT id, name, age FROM users")
rows = cur.fetchall()

for r in rows:
    print(r)

conn.close()

Output: prints tuples like (1, 'Asha', 18) (depends on your data).

4.3 Advanced Example — update + delete

Python
update/delete
import sqlite3

conn = sqlite3.connect("app.db")
cur = conn.cursor()

# Update age for a specific name
cur.execute("UPDATE users SET age=? WHERE name=?", (21, "Asha"))

# Delete user by name
cur.execute("DELETE FROM users WHERE name=?", ("Asha",))

conn.commit()
conn.close()
print("Updated and deleted")

Output: Updated and deleted

6) Memory Visualization

When you do fetchall(), Python loads all matching rows into memory. For large tables, you should fetch in smaller pieces (concept).

Method Memory behavior
fetchone() one row at a time
fetchall() loads all rows into a list
execute() runs query and prepares results

15) Debugging Practice

Case 1: wrong SQL syntax

Python
bug
cur.execute("SELECT name age FROM users")

Fix: SQL needs commas: SELECT name, age ...

Case 2: forgetting commit

Python
bug
cur.execute("INSERT INTO users(name, age) VALUES(?, ?)", ("Ravi", 20))
# missing conn.commit()
conn.close()

Fix: call conn.commit() after writes.

Case 3: string concatenation SQL injection risk

Python
bug
name = "Ravi'; DROP TABLE users; --"
cur.execute("SELECT * FROM users WHERE name='" + name + "'")

Fix: use WHERE name=? and pass (name,).

16) Common Errors

  • sqlite3.OperationalError: SQL syntax or table/column missing.
  • sqlite3.IntegrityError: constraint failures (concept).
  • ProgrammingError: using cursor/connection incorrectly.
  • Data mismatch: inserting wrong types (concept).

17) Best Practices

  • Use parameterized queries (?) always with user input.
  • Create tables with IF NOT EXISTS for learning scripts.
  • Call commit() after insert/update/delete.
  • Select only needed columns.
  • Fetch carefully: avoid fetchall() for huge result sets (concept).

Key explanations

  • Database: stores data persistently.
  • CRUD: create, read, update, delete.

  • Parameterized queries: prevent SQL injection by separating code from data.
