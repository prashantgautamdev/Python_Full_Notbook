1) Introduction

A Python script can do more than compute. It can talk to the operating system.

In this chapter you will learn the basics of OS programming: working with paths, checking files, and running commands.

2) Theory

2.1 The os module

os gives functions for filesystem and OS-level tasks. Examples: check if a path exists, create directories, list files.

2.2 Paths: why you should care

A path is like an address to a file. Windows uses backslashes, but Python provides helpers to avoid mistakes.

2.3 Environment variables

Some values are stored by the OS, like PATH or API keys. Python can read them using os.environ.

2.4 subprocess concept

When you need to run a command-line program from Python, you use subprocess.

⚠ Warning: security

Never build shell commands using untrusted user input. In production you must sanitize inputs.

3) Syntax

Python
list files
import os

files = os.listdir(".")
print(files)
Python
path helpers
import os

p = os.path.join("folder", "data.txt")
print(p)

print(os.path.exists(p))
Python
environment variables
import os

print(os.environ.get("PATH", "PATH not set"))
Python
subprocess (safe form)
import subprocess

# Example: run a harmless command (works differently by OS)
result = subprocess.run(["python", "--version"], capture_output=True, text=True)
print("stdout:", result.stdout.strip())
print("stderr:", result.stderr.strip())

Time: depends on OS operations and command runtime • Space: depends on data collected (like stdout).

Alternative: use filesystem libraries or higher-level tools; but os is the simplest standard approach.

4) Examples

4.1 Easy Example — check if a file exists

Python
exists + message
import os

path = "notes.txt"
if os.path.exists(path):
    print("Found", path)
else:
    print("Missing", path)

Output: prints Found or Missing depending on your files.

4.2 Medium Example — create a folder and write a file

Python
mkdir + write
import os

folder = "output_notes"

# exist_ok=True avoids error if the folder already exists
os.makedirs(folder, exist_ok=True)

file_path = os.path.join(folder, "hello.txt")

with open(file_path, "w", encoding="utf-8") as f:
    f.write("Hello from OS programming!\n")

print("Wrote:", file_path)

4.3 Advanced Example — run a command and read output

Python
subprocess output
import subprocess

cmd = ["python", "-c", "print('hi from child process')"]
res = subprocess.run(cmd, capture_output=True, text=True)

print("Exit code:", res.returncode)
print("Output:", res.stdout.strip())

Output: shows the printed message.

6) Memory Visualization

When you call os.listdir(), Python loads a list of filenames into memory. When you call subprocess.run(..., capture_output=True), Python stores stdout/stderr strings.

Operation What may use memory
os.listdir filename list
open() file content (if using read())
subprocess captured stdout/stderr

15) Debugging Practice

Python
bug
import os

Fix: ensure you run from correct folder or use absolute paths.

Case 2: missing file permissions

Python
bug
with open("/protected/out.txt", "w") as f:
    f.write("hi")

Fix: choose a writable folder and handle PermissionError.

Python
bug

print(res.returncode)
print(res.stderr)

Fix: check returncode and view stderr.

16) Common Errors

  • FileNotFoundError: path doesn’t exist.
  • PermissionError: OS blocks access.
  • OSError: general OS-related failure.
  • subprocess errors: invalid command or non-zero return code.

17) Best Practices

  • Use os.path.join instead of hardcoded separators.
  • Create folders with os.makedirs(..., exist_ok=True).
  • Prefer list form in subprocess.run instead of building shell strings.
  • Use with open() for safe file writes.
  • Handle OS errors with try/except when filenames come from users.
