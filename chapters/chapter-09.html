1) Introduction

In earlier chapters, you wrote code once. In real projects, you repeat the same logic many times.

⭐ Remember

A function is a reusable block of code. It has a name, can take inputs, and may return an output.

2) Theory

2.1 Why functions?

  • Reuse: write logic once and call it many times.
  • Clarity: hide complexity behind a simple name.
  • Testing: test function separately.

2.2 Parameters and arguments

  • Parameters: names in the function definition.
  • Arguments: actual values you pass when calling.

2.3 Return vs print

print() shows output on screen. return sends a value back to the caller.

⚠ Warning

If a function doesn’t have return, Python returns None automatically.

3) Syntax

Python
def + call
# Function definition
def add(a, b):
    # a and b are parameters
    result = a + b
    return result

# Function call
x = add(2, 3)
print(x)

Time complexity: O(1) for simple operations • Space: O(1) extra.

Alternative: do math directly, but functions improve reuse.

4) Examples

4.1 Easy Example — greeting function

Python
returns a string
def greet(name):
    message = "Hello, " + name
    return message

print(greet("Ravi"))

Output: Hello, Ravi

4.2 Medium Example — converting input

Python
multiple calls
def square(n):
    return n * n

nums = [1, 2, 3]
for v in nums:
    print(square(v))

Output: 1 4 9 (each new line)

Time: O(m) for m numbers processed • Space: O(1) extra.

4.3 Advanced Example — function that doesn’t return

Python
returns None
def show_total(a, b):
    total = a + b
    print("Total:", total)

x = show_total(2, 3)
print("Function returned:", x)

Output: Total: 5 then Function returned: None

6) Memory Visualization

Each function call creates a new “local memory” for parameters and local variables. After the function ends, local values are discarded.

Concept Where Example
Parameters Inside function scope a, b
Local variables Inside function scope result
Returned value Back to caller 5

🧠 Memory trick

Function scope is like a temporary room. When you leave, the room is cleared.

7) Dry Run

Python
dry run add()
def add(a, b):
    result = a + b
    return result

x = add(4, 7)
print(x)
Step Action Value
1 Call add(4,7) a=4, b=7
2 Compute result=a+b result=11
3 Return result x=11

Output: 11

Time: O(1) • Space: O(1)

Alternative: compute directly without defining a function (less reusable).

15) Debugging Practice

Case 1: forgot return

Python
bug
def add(a,b):
    return a+b

def bad_add(a,b):
    result = a+b

x = bad_add(2,3)
print(x)

Fix: add return result. Without return, Python returns None.

Case 2: wrong parameter names

Python
bug
def greet(name):
    return "Hi " + person

Issue: person variable doesn’t exist.
Fix: use name.

Case 3: calling with wrong number of arguments

Python
bug
def add(a,b):
    return a+b

print(add(5))

Fix: call with two arguments: add(5,7).

16) Common Errors

  • NameError: using a variable not defined.
  • TypeError: wrong types or wrong number of arguments.
  • NoneType error: trying to use a function that returned None.
  • SyntaxError: missing colon after def.

17) Best Practices

  • Name functions with verbs (e.g., add, calculate_total).
  • Prefer return values over printing in business logic.
  • Keep functions small and single-purpose.
  • Test functions with multiple inputs.
  • Document parameters with comments.

18) Summary

Functions let you reuse code. You define a function with def, pass arguments as parameters, and optionally return a value with return. If you don’t return, Python returns None.
