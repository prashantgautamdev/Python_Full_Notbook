1) Introduction

Python is a programming language. A programming language is a set of rules that lets you write instructions for a computer.

Python is famous because it is readable. Many beginners choose Python first because the code looks close to normal English.

⭐ Remember Box

Python uses indentation (spaces at the beginning of lines) to group code. That means the “shape” of your code matters.

2) Theory

📚 Big Picture

Think of a program like a recipe. You provide ingredients (data), and the program follows steps to produce a result.

2.1 What happens when you run Python?

When you run a Python program, Python reads the code line by line and executes it. If something is wrong, Python raises an error. Errors are not “failures”. They are clues that help you fix your instructions.

2.2 Program building blocks

  • Variables store values.
  • Types tell the kind of value (like numbers or text).
  • Statements tell Python what to do.
  • Expressions produce a value.
  • Functions are reusable actions.

⚠ Warning

Don’t worry if concepts feel new. The goal here is to build a mental model. You will learn the details step by step in later chapters.

3) Syntax

Syntax means the exact rule format that Python requires. If syntax is wrong, Python cannot understand your code.

3.1 The simplest program

Python
Statement • output
# This line is a comment. Python ignores it.
# A comment helps humans understand code.

print("Hello, Python!")

💡 Output prediction

If you run this program, you should see:

  • Hello, Python!

The print() function sends text to the screen.

Time complexity: O(1) (it prints one fixed message).
Space complexity: O(1) (fixed output).

Alternative approach: use string variables and then print the variable.

4) Examples

4.1 Easy Example — variables and types

Python
Variables • print
# Variables store values.
name = "Asha"          # name is a text value (string)
age = 12               # age is a number (integer)

print(name)
print(age)

Output:

  • Asha
  • 12

Time: O(1) • Space: O(1)

Alternative approach: combine them in one print using f-strings (later chapters will cover f-strings deeply).

4.2 Medium Example — simple input/output thinking

Python
Input-like flow (no real input)
# We can simulate "input" by using a variable.
# In later chapters, you will use input() to read from keyboard.

user_name = "Ravi"
message = "Welcome, " + user_name

print(message)

Output: Welcome, Ravi

Time: O(n) where n is length of the resulting string • Space: O(n)

Alternative approach: use f"Welcome, {user_name}" (Python 3).

4.3 Advanced Example — functions as mental tools

Python
Function call • reuse
# Example of calling a built-in function.
# range() is a generator of numbers.

total = 0
for x in range(5):
    total += x

print(total)

Output: 10 (0+1+2+3+4)

Time: O(n) • Space: O(1) extra space (loop variables).

Alternative approach: use sum(range(5)) (built-in aggregation).

8) Built-in Functions

These are functions that come with Python. You can use them immediately.

Common built-ins in this chapter

  • print(): shows values.
  • range(): generates numbers for loops.
  • sum(): totals numbers (alternative to loops).
Python
Built-in: sum()
# Sum numbers quickly using sum().

print(sum(range(5)))

Output: 10

Time: O(n) • Space: O(1) extra.

Alternative approach: use a for-loop accumulator (shown earlier).

9) Methods

Methods are functions that belong to an object. In Python, strings have methods. (We will go deeper later, but here is a taste.)

Python
String method: upper()
text = "python"
print(text.upper())

Output: PYTHON

Time: O(n) • Space: O(n) for new string.

Alternative approach: use text.capitalize() or text.title().

16) Common Errors

  • NameError: using a variable that was never assigned.
  • SyntaxError: code format doesn’t match Python rules.
  • TypeError: using the wrong kind of value (e.g., string + int).
  • IndentationError: loop/function block indentation is wrong.
  • ValueError: converting wrong input to a type (later chapters).

17) Best Practices

  • Run small code often.
  • Use comments to explain intent.
  • Name variables clearly: age, user_name.
  • Prefer simple output with print() while learning.
  • Read error messages carefully; they usually tell the line number.

18) Summary

Python is readable and beginner-friendly. A program has a flow: data in → processing → output. Variables store values. Types matter. You can use built-in functions like print() and range().
