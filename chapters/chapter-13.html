🎯 Learning Objectives

By the end of this chapter, you will be able to:

  • Create dictionaries using {} with key: value pairs.
  • Access values by key.
  • Update values and add new key/value pairs.
  • Loop through keys, values, and items.
  • Use common dictionary methods like get().

1) Introduction

Sometimes you want to store information in a way that you can look it up quickly. For example:

  • Name → age
  • Student ID → marks
  • Product → price

Dictionaries store data using keys and values.

2) Theory

2.1 Key/value pairs

A dictionary maps a key to a value.

2.2 Keys must be unique

If you assign a value to an existing key, it overwrites the old value.

⚠ Warning: missing keys

Accessing a dictionary with a key that doesn’t exist raises KeyError. Use get() when you want safe access.

2.3 Looping concept

  • Looping directly gives keys.
  • Use .values() for values.
  • Use .items() for key/value pairs.

3) Syntax

Python
create + access
student = {"name": "Ravi", "age": 20}

print(student["name"])
print(student["age"])
Python
update + add
student = {"name": "Ravi", "age": 20}

student["age"] = 21     # update
student["city"] = "Delhi"  # add new

print(student)

Time complexity: lookups are fast on average • Space: proportional to number of items.

Alternative: use parallel lists or classes (advanced), but dict is simplest for mapping.

4) Examples

4.1 Easy Example — age book

Python
lookup
ages = {"Asha": 18, "Ravi": 20}

print("Asha age:", ages["Asha"])

4.2 Medium Example — get() safe access

Python
get()
ages = {"Asha": 18, "Ravi": 20}

print("Kiran age:", ages.get("Kiran", "Not found"))

4.3 Advanced Example — loop with items()

Python
items loop
student = {"name": "Maya", "age": 22, "city": "Pune"}

for key, value in student.items():
    print(key, "=", value)

15) Debugging Practice

Case 1: KeyError

Python
bug
d = {"a": 1}
print(d["b"])

Fix: use d.get("b", "default") or check key existence.

Case 2: wrong syntax (missing colon)

Python
bug
d = {"a" 1}

Fix: dict uses key: value.

Case 3: overwriting values unintentionally

Python
bug
d = {"x": 1}
d["x"] = 2
print(d)

Reason: keys are unique. Update overwrites.

16) Common Errors

  • KeyError: key doesn’t exist.
  • SyntaxError: missing colon or quotes.
  • TypeError: using wrong key type (concept).
  • Logic error: expecting ordering or wrong key name.

17) Best Practices

  • Use get() when key might be missing.
  • Use descriptive keys (e.g., "student_id").
  • Validate input before using as key.
  • Prefer loops over values/items for reports.
  • Avoid relying on printed order.

18) Summary

Dictionaries map keys to values. You create them with {key: value} pairs. You access with d[key], update/add with assignment, and loop with keys(), values(), and items(). Use get() to avoid KeyError.
