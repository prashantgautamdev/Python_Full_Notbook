1) Introduction

Command-line programs print text and then exit. Desktop apps show buttons, forms, and screens.

In Python, Tkinter is the standard beginner-friendly library for building desktop GUIs. It comes with most Python installations.

2) Theory

2.1 What is Tkinter?

Tkinter is a library that gives you GUI widgets. Widgets are things like labels, text boxes, and buttons.

2.2 Event-driven programming

Your program starts a window and then waits. When the user clicks a button, the program runs a callback function.

2.3 pack vs grid (concept)

  • pack: stacks widgets (easy for simple layouts).
  • grid: places widgets in rows and columns (best for forms).

⚠ Warning

GUI code can freeze if you run long tasks inside callbacks. For heavy work, use background threads (later chapters).

3) Syntax

Python
minimal Tkinter app
import tkinter as tk

root = tk.Tk()               # create main window
root.title("My App")

label = tk.Label(root, text="Hello GUI!")
label.pack()                # place the widget

root.mainloop()             # start event loop (wait for user)

Output: a window that shows “Hello GUI!”.

Time: O(1) to create widgets • Space: O(number of widgets).

Alternative: use another GUI framework (PyQt/Tkinter alternatives). Tkinter is chosen here for simplicity.

4) Examples

4.1 Easy Example — Label + Button

Python
button updates label
import tkinter as tk

root = tk.Tk()
root.title("Button Example")

label = tk.Label(root, text="Press the button")
label.pack(pady=10)

def on_click():
    # callback runs when user clicks
    label.config(text="Button clicked!")

btn = tk.Button(root, text="Click me", command=on_click)
btn.pack(pady=10)

root.mainloop()

Output: clicking “Click me” changes the label text.

4.2 Medium Example — Entry + Echo

Python
input + output
import tkinter as tk

root = tk.Tk()
root.title("Echo App")

prompt = tk.Label(root, text="Type something:")
prompt.grid(row=0, column=0, padx=8, pady=8)

entry = tk.Entry(root, width=30)
entry.grid(row=0, column=1, padx=8, pady=8)

result = tk.Label(root, text="(nothing yet)")
result.grid(row=1, column=0, columnspan=2, padx=8, pady=8)

def echo():
    text = entry.get()           # read the Entry widget value
    result.config(text=f"You typed: {text}")

btn = tk.Button(root, text="Echo", command=echo)
btn.grid(row=2, column=0, columnspan=2, pady=10)

root.mainloop()

Output: type text, click Echo, see the same text.

4.3 Advanced Example — simple validator

Python
reject empty input
import tkinter as tk

root = tk.Tk()
root.title("Validator")

entry = tk.Entry(root, width=30)
entry.pack(pady=10)

msg = tk.Label(root, text="Enter your name")
msg.pack(pady=5)

def submit():
    name = entry.get().strip()
    if not name:
        msg.config(text="⚠ Name cannot be empty", fg="#d93025")
        return
    msg.config(text=f"✅ Hello, {name}", fg="#0f9d58")

btn = tk.Button(root, text="Submit", command=submit)
btn.pack(pady=10)

root.mainloop()

Output: if empty → warning message, otherwise greeting.

15) Debugging Practice

Case 1: window closes immediately

bug
forgot mainloop
# missing root.mainloop()
import tkinter as tk
root = tk.Tk()

Fix: always call root.mainloop() at the end.

Case 2: button does nothing

bug
command missing
btn = tk.Button(root, text="Click")
# missing command=on_click

Fix: pass command=on_click to connect the callback.

Case 3: value always empty

bug
using Entry wrong
# using entry.text instead of get
text = entry.text

Fix: use entry.get().

16) Common Errors

  • AttributeError: using wrong widget property (use get(), config()).
  • NameError: callback function not defined before using.
  • TypeError: calling widget methods with wrong parameters.
  • UI freeze: long running work inside callbacks.

17) Best Practices

  • Keep callbacks small. Move logic into helper functions.
  • Validate input before using it.
  • Update only the widgets you need.
  • Use grid for forms, pack for simple stacking.
  • Avoid heavy work inside callbacks to prevent freezing.
