1) Introduction

Humans see images easily. Computers need images turned into numbers.

Computer vision studies how to do that.

2) Theory

2.1 Pixels and channels

A digital image is a grid. Each cell is a pixel.

  • Grayscale: one value per pixel.
  • RGB: three values per pixel (red, green, blue).

2.2 Grayscale conversion (concept)

Grayscale is often computed as a weighted sum of RGB values. Example concept: gray ≈ 0.299R + 0.587G + 0.114B.

2.3 Convolution filtering (concept)

Filters look at a small neighborhood (like 3×3). A kernel multiplies neighbors and sums them.

2.4 Edge detection idea

Edges are where pixel values change quickly. A kernel can highlight those changes.

⚠ Warning

Filtering can create noise or lose details. Always check output visually and test on multiple images.

3) Syntax

Python
tiny edge detector (kernel concept)
# This is a teaching example using a simple 3x3 kernel idea.
# We'll compute a horizontal gradient using a Sobel-like kernel concept.


def convolve_2d(grid, kernel):
    """
    grid: list of lists (2D) of numbers
    kernel: 3x3 list of lists of numbers

    Time complexity: O(H*W*kH*kW) where k is kernel size (constant 3).
    Space complexity: O(H*W) for output.
    """
    H = len(grid)
    W = len(grid[0])

    kH = len(kernel)
    kW = len(kernel[0])
    pad_h = kH // 2
    pad_w = kW // 2

    # Output grid (same size for simplicity with zero padding)
    out = [[0 for _ in range(W)] for _ in range(H)]

    for r in range(H):
        for c in range(W):
            s = 0
            for kr in range(kH):
                for kc in range(kW):
                    rr = r + (kr - pad_h)
                    cc = c + (kc - pad_w)

                    val = grid[rr][cc] if (0 <= rr < H and 0 <= cc < W) else 0
                    s += val * kernel[kr][kc]
            out[r][c] = s

    return out


# Example 2D "image" (grayscale values)
img = [
    [0, 0, 0, 0, 0],
    [0, 10, 10, 10, 0],
    [0, 10, 50, 10, 0],
    [0, 10, 10, 10, 0],
    [0, 0, 0, 0, 0],
]

# Simple edge kernel concept (detect changes)
kernel = [
    [-1, -1, -1],
    [-1,  8, -1],
    [-1, -1, -1]
]

edges = convolve_2d(img, kernel)
for row in edges:
    print(row)

Output: numbers showing strong response near changes.

Alternative: use OpenCV or scikit-image for real filters.

7) Dry Run

Dry run
center pixel response
Using kernel with center weight 8 and neighbors -1 (concept).
For the center of the image, neighbors are ~10 and center is 50.

response ≈ 8*50 + (-1)*(8 neighbors * 10)
         ≈ 400 - 80
         ≈ 320 (large positive)

So the center becomes a strong "edge/feature" response.

Time: O(H*W) with constant kernel • Space: O(H*W).

15) Debugging Practice

Case 1: index error at boundaries

bug
no bounds check
# rr/cc may go out of bounds
val = grid[rr][cc]

Fix: add padding logic when rr/cc invalid.

Case 2: output looks shifted

bug
wrong padding alignment
# pad_h/pad_w computed incorrectly
pad_h = kH

Fix: use kH//2 and kW//2.

Case 3: everything is zero

bug
threshold too high
mask = out > 10000

Fix: check min/max of output before threshold.

16) Common Errors

  • IndexError: accessing out-of-range pixels.
  • ZeroDivisionError: if normalization divides by 0 (concept).
  • TypeError: passing non-numeric grid values.
  • ValueError: kernel not odd sized.

17) Best Practices

  • Start with tiny arrays you can reason about.
  • Print min/max and a few output rows for sanity checks.
  • Document kernel meaning and units.
  • Normalize carefully after filtering (if needed).
  • Validate output dimensions and boundary behavior.

22) Answer Key (Explain every answer)

Key explanations

  • Pixel: a single value at a coordinate.
  • RGB: 3 channels (R,G,B) per pixel.
  • Grayscale: one value representing intensity.
  • Kernel: small matrix of weights for filtering.
  • Convolution: apply kernel to neighborhoods to compute response.
  • Edges: where values change quickly.
  • Padding: boundary handling to keep convolution valid.
  • Thresholding: convert response to a binary mask.
  • Feature map: output of filters showing detected patterns.
