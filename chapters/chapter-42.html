1) Introduction

If Python programs always used “one way” to store and search data, everything would be slow.

Data structures and algorithms teach you how to choose the right approach so your solution scales.

2) Theory

2.1 Data structures: where data lives

  • Array/list: fast index access for lists.
  • Stack: Last-In, First-Out (LIFO).
  • Queue: First-In, First-Out (FIFO).
  • Tree: parent-child hierarchy (like folders).
  • Graph: nodes connected by edges (like maps).

2.2 Algorithms: how you transform data

  • Search: find an element.
  • Sort: order elements to enable faster operations.
  • Traverse: visit data structure nodes systematically.

2.3 Complexity (Big-O concept)

Big-O tells how time grows when input size n becomes large. Example idea:

Algorithm Typical time Meaning
Linear search O(n) checks each item once
Binary search O(log n) halves search space each step
Bubble sort O(n²) nested comparisons
Merge sort O(n log n) divide + combine

⚠ Warning

Big-O ignores constant factors and real-world hardware effects. But it is still the best tool for comparing scaling.

4) Examples

4.1 Easy Example — linear search

Python
find value in list
def linear_search(arr, target):
    """Return index of target or -1 if not found."""
    # Time: O(n)
    # Space: O(1)
    for i in range(len(arr)):
        # Compare current element to target
        if arr[i] == target:
            return i
    return -1

arr = [4, 10, 3, 7]
print(linear_search(arr, 7))  # Output: 3
print(linear_search(arr, 99)) # Output: -1

4.2 Medium Example — binary search

Python
requires sorted array
def binary_search(arr, target):
    """Return index of target or -1 if not found."""
    # Time: O(log n)
    # Space: O(1)
    left = 0
    right = len(arr) - 1

    while left <= right:
        mid = (left + right) // 2

        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            left = mid + 1
        else:
            right = mid - 1

    return -1

arr = [1, 3, 4, 7, 10]
print(binary_search(arr, 7))  # Output: 3
print(binary_search(arr, 2))  # Output: -1

4.3 Advanced Example — selection sort (teaching)

Python
sort by repeatedly selecting min
def selection_sort(arr):
    """Return a new sorted list using selection sort."""
    # Time: O(n^2)
    # Space: O(1) extra if in-place; here we sort copy

    a = arr[:]  # make a copy so original stays same

    n = len(a)
    for i in range(n):
        # Assume the minimum is at position i
        min_idx = i

        # Find the smallest element in the rest
        for j in range(i + 1, n):
            if a[j] < a[min_idx]:
                min_idx = j

        # Swap minimum found with element at i
        a[i], a[min_idx] = a[min_idx], a[i]

    return a

nums = [5, 2, 9, 1]
print(selection_sort(nums))  # Output: [1, 2, 5, 9]

15) Debugging Practice

Case 1: off-by-one in binary search

bug
mid boundaries
while left < right:  # wrong can skip answer

Fix: for typical binary search use left <= right.

Case 2: forgot to update queue

bug
no visited update
# BFS revisits nodes forever

Fix: add to visited when enqueuing.

Case 3: selection sort swaps wrong indices

bug
swap with i always
a[i], a[i] = a[i], a[i]  # no effect

Fix: swap with min_idx.

16) Common Errors

  • IndexError: using invalid mid or loop boundaries.
  • Infinite loop: missing visited update in graph search.
  • Logic bug: binary search on unsorted array.
  • Performance issue: using O(n²) where O(n log n) is possible.

17) Best Practices

  • Choose data structure based on operations needed (search/insert/remove).
  • Use visited sets for graph/tree traversal.
  • Test edge cases: empty list, single element, missing target.
  • Reason about complexity before implementing heavy logic.
  • Use Python built-ins when they match your need.

22) Answer Key (Explain every answer)

Key explanations

  • Stack: LIFO (push last, pop last).
  • Queue: FIFO (enqueue back, dequeue front).
  • Linear search: checks each element until found.
  • Binary search: halves search space each step.
  • Selection sort: selects the minimum and swaps into place.
  • Big-O: describes how runtime grows with n.
  • Graph traversal: visits nodes while tracking visited to avoid loops.
