1) Introduction

Sometimes your code runs into problems. Example: a user types abc instead of a number.

Python can show an error message and stop the program. Exception handling lets your program recover gracefully.

2) Theory

2.1 What is an exception?

An exception is an error that happens during execution. It can be caught and handled.

2.2 try / except

try contains code that might fail. except handles the failure.

⭐ Remember

If no error happens, except does nothing. If an error happens, Python jumps to the matching except block.

2.3 else and finally

  • else runs only if no exception occurred.
  • finally runs always (cleanup).

⚠ Warning: don’t hide bugs

Catching every exception can hide real errors. Catch the specific error you expect.

3) Syntax

Python
try/except
try:
    x = int(input("Enter a number: "))
except ValueError:
    print("That was not a valid integer.")
Python
else + finally
try:
    data = "123"
    value = int(data)
except ValueError:
    print("Bad data")
else:
    print("Converted successfully")
finally:
    print("Cleanup always runs")

Time complexity: depends on code inside try • Space: depends on stored data.

Alternative: validate input before converting to reduce exceptions.

4) Examples

4.1 Easy Example — safe int conversion

Python
ValueError
text = "abc"

try:
    n = int(text)
    print("Number:", n)
except ValueError:
    print("Cannot convert to int")

Output: Cannot convert to int

4.2 Medium Example — handle division by zero

Python
ZeroDivisionError
a = 10
b = 0

try:
    result = a / b
    print("Result:", result)
except ZeroDivisionError:
    print("Cannot divide by zero")

Output: Cannot divide by zero

4.3 Advanced Example — finally cleanup

Python
always runs
try:
    f = open("example.txt", "r")
    content = f.read()
    print(content)
except FileNotFoundError:
    print("File not found")
finally:
    try:
        f.close()
    except Exception:
        pass

Output: either file content or file not found, and cleanup happens.

16) Common Errors

  • SyntaxError: missing colon after except.
  • NameError: using a variable that was never assigned because conversion failed.
  • KeyError: accessing dict without checking key existence (use get or handle exceptions).
  • Logic error: catching too broad exceptions and masking real bugs.

17) Best Practices

  • Catch specific exceptions you expect.
  • Use input validation when you can.
  • Provide helpful, user-friendly messages.
  • Use finally for cleanup.
  • Avoid catching everything with except Exception.

18) Summary

Exceptions are runtime errors. Use try to attempt risky code, except to handle specific problems, else for the success path, and finally for cleanup always.

19) Cheat Sheet

Need Write Meaning
Handle risky code try: execute risky operations
Handle error except ValueError: catch specific errors
Success-only else: run when no error
Cleanup always finally: runs always
