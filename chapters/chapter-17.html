🎯 Learning Objectives

By the end of this chapter, you will be able to:

  • Create classes using class.
  • Create objects (instances) from classes.
  • Use attributes to store data.
  • Use methods to define behavior.
  • Understand self and the constructor idea.

1) Introduction

If your program uses many “things” (like users, students, orders, vehicles), OOP helps you organize them.

Instead of passing lots of variables around, you bundle data + behavior into classes.

2) Theory

2.1 Class and object

A class is a blueprint. An object is created from that blueprint.

2.2 Attributes and methods

  • Attributes store data (like name, age).
  • Methods perform actions (like greet()).

2.3 The self keyword

self represents the current object. It lets a method read/write that object’s attributes.

⚠ Warning

Inside class methods, you must use self.attribute to store attributes. Don’t write just attribute = ... (that becomes a local variable).

3) Syntax

Python
basic class
class Person:
    # This is like a blueprint.
    pass
Python
constructor + method
class Person:
    def __init__(self, name, age):
        # Store data in attributes.
        self.name = name
        self.age = age

    def greet(self):
        # Use stored attributes.
        return f"Hello, I am {self.name} and I am {self.age}."

p = Person("Ravi", 20)
print(p.greet())

Time complexity: depends on method logic • Space: object holds attributes.

Alternative: procedural style (functions + dicts) is possible, but OOP scales better for complex systems.

4) Examples

4.1 Easy Example — student class

Python
attributes + greet
class Student:
    def __init__(self, name, marks):
        self.name = name
        self.marks = marks

    def status(self):
        if self.marks >= 40:
            return "Pass"
        return "Fail"

s = Student("Asha", 55)
print(s.status())

Output: Pass

4.2 Medium Example — multiple objects

Python
loop over objects
class Product:
    def __init__(self, name, price):
        self.name = name
        self.price = price

    def price_tag(self):
        return f"{self.name}: {self.price}" 

items = [
    Product("Pen", 10),
    Product("Notebook", 50)
]

for item in items:
    print(item.price_tag())

Output: Pen: 10 then Notebook: 50 (order as listed).

4.3 Advanced Example — method returns computed result

Python
simple business logic
class Rectangle:
    def __init__(self, width, height):
        self.width = width
        self.height = height

    def area(self):
        return self.width * self.height

    def perimeter(self):
        return 2 * (self.width + self.height)

r = Rectangle(5, 7)
print("Area:", r.area())
print("Perimeter:", r.perimeter())

Output: Area: 35 and Perimeter: 24

7) Dry Run

Python
constructor dry run
class Person:
    def __init__(self, name, age):
        self.name = name
        self.age = age

p = Person("Ravi", 20)
print(p.name)
print(p.age)
Step Line What changes
1 p=Person(...) Create new object instance p
2 self.name=name Store self.name="Ravi"
3 self.age=age Store self.age=20
4 print(p.name) prints Ravi

Time: O(1) • Space: stores attributes for object.

Alternative: use dicts like {"name":...,"age":...} (works but OOP gives methods for behavior).

9) Methods

In OOP, “methods” are functions inside a class. The term method is important because it usually operates on self.

💡 Tip

If you think “this function needs the object data,” it belongs in the class as a method.

15) Debugging Practice

Case 1: forgetting self

Python
bug
class Person:
    def __init__(self, name):
        name = name

p = Person("Ravi")
print(p.name)

Fix: use self.name = name.

Case 2: calling method wrong

Python
bug
class A:
    def hello(self):
        return "Hi"

print(A.hello())

Fix: create object first: a=A(); print(a.hello()).

Case 3: local variable vs attribute

Python
bug
class Counter:
    def __init__(self):
        self.count = 0

    def inc(self):
        count = self.count + 1

c = Counter()
c.inc()
print(c.count)

Fix: update self.count = count or directly self.count += 1.

16) Common Errors

  • AttributeError: accessing attribute not stored on object.
  • TypeError: calling a method without creating object (missing self).
  • NameError: using variables not defined inside methods.
  • Logic error: updating local variable instead of self. attribute.

17) Best Practices

  • Name classes using nouns: Person, Rectangle.
  • Use methods to group behavior with data.
  • Always store instance data as self.x inside __init__.
  • Return values from methods instead of printing inside methods (clean design).
  • Keep methods short and focused.
