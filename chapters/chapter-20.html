1) Introduction

Dates and times are everywhere: birthdays, deadlines, logs, reminders, and timestamps. Python needs special tools to handle them correctly.

In this chapter you will learn the standard library module datetime, and how to do safe time math.

🌍 Real-life example

A fitness app stores your activity time stamps. It needs to show “how many hours ago” and sort events by time.

2) Theory

2.1 datetime vs date vs time

  • date: year, month, day.
  • time: hour, minute, second, microsecond.
  • datetime: both date and time together.

2.2 timedelta = duration

timedelta represents a duration like “3 days 4 hours”. You usually get it by subtracting two datetime objects.

2.3 Formatting and parsing

Dates are often stored as strings (like from a CSV). Python needs to convert them into datetime objects (parse), and later convert back to strings (format).

⚠ Warning: time zones (concept)

This chapter focuses on basic usage without deep time-zone complexity. Real systems must handle time zones carefully.

Alternative: use plain strings, but you lose correct ordering and time math.

4) Examples

4.1 Easy Example — current timestamp

Python
now
from datetime import datetime

now = datetime.now()
print("Now:", now)

Output: prints the current date and time (varies each run).

4.2 Medium Example — parse + format

Python
CSV string → datetime
from datetime import datetime

text = "2026-07-13"
d = datetime.strptime(text, "%Y-%m-%d")

print("Parsed:", d)
print("Formatted:", d.strftime("%d %B %Y"))

Output: Parsed then a human readable date like 13 July 2026 (month name depends on locale).

Time: O(len(text)) for parsing • Space: O(1) object overhead.

4.3 Advanced Example — time difference with timedelta

Python
subtract datetimes
from datetime import datetime, timedelta

start = datetime(2026, 7, 1, 9, 0, 0)
end = datetime(2026, 7, 1, 12, 30, 0)

elapsed = end - start
print("Elapsed:", elapsed)

# Add duration
bonus = start + timedelta(hours=2)
print("Two hours later:", bonus)

Output: elapsed shows something like 3:30:00 and “two hours later” prints a later time.

14) Mini Project

🧪 Mini Project: Event Countdown (Text Output)

Create a script:

  • Take an event datetime string like 2026-07-20 18:00:00.
  • Compare it to datetime.now().
  • Print: “Event is in X days” or “Event time passed by X days”.
Python
starter
from datetime import datetime

event_text = input("Enter event (YYYY-MM-DD HH:MM:SS): ").strip()

event_dt = datetime.strptime(event_text, "%Y-%m-%d %H:%M:%S")
now = datetime.now()

diff = event_dt - now

if diff.total_seconds() >= 0:
    print("Event is in", diff.days, "day(s)")
else:
    print("Event already passed by", abs(diff.days), "day(s)")

15) Debugging Practice

Case 1: format mismatch

Python
bug
from datetime import datetime

text = "2026-07-13"
dt = datetime.strptime(text, "%Y/%m/%d")
print(dt)

Fix: match separators. If input uses -, format must use - too.

Case 2: using %m vs %M

Python
bug
from datetime import datetime

dt = datetime(2026, 7, 13, 10, 30, 0)
print(dt.strftime("%m"))  # month
print(dt.strftime("%M"))  # minute

Fix: remember %m=month and %M=minute.

Case 3: calling .group() on datetime (concept mistake)

Python
bug
from datetime import datetime

text = "2026-07-13"
dt = datetime.strptime(text, "%Y-%m-%d")
print(dt.group(1))

Fix: group() belongs to regex match objects. For datetime use field access like dt.year.

16) Common Errors

  • ValueError: strptime format doesn’t match the input string.
  • TypeError: mixing incompatible types in arithmetic (e.g., adding int instead of timedelta).
  • Logic error: incorrect format directives (%m vs %M, wrong order).
  • Timezone logic issues: naive datetime comparisons can be misleading (concept).

17) Best Practices

  • Always parse strings into datetime before doing time math.
  • Use strftime/strptime with correct directives; double-check placeholders.
  • Keep formatting simple for learning; avoid complex one-line prints.
  • When parsing user data, wrap in try/except and show a helpful message.
  • Be careful around time zones in real projects.
