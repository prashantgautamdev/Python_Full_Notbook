1) Introduction

You already used strings in earlier chapters. Now we will treat strings as a real data type with tools.

🧠 Memory trick

Think of a string like a line of letters. Each letter has a position (index).

2) Theory

2.1 Creating strings

Strings can be written using single quotes (' ') or double quotes (" ").

2.2 Indexing

Index starts at 0 for the first character. Negative indexes count from the end.

2.3 Slicing

Slicing lets you take a part of a string: s[start:end]. The end index is excluded.

⚠ Warning: immutability

Strings cannot be changed in-place like lists. When you use methods like upper(), Python creates a new string.

2.4 Useful string tasks

  • Cleaning spaces: strip()
  • Changing case: upper(), lower()
  • Finding content: find() (concept preview)
  • Splitting words: split() (later used heavily)

3) Syntax

Python
quotes, indexing, slicing
s = "Python"

print(s[0])   # first letter
print(s[3])   # 4th letter
print(s[-1])  # last letter

print(s[0:3]) # "Pyt"
print(s[2:5]) # "tho"
print(s[:2])  # from start to index 2 (excluded)
print(s[3:])  # from 3 to end

Time complexity: O(k) where k is slice length • Space: O(k) for new strings.

Alternative approach: iterate character by character (use loops later; concept now).

4) Examples

4.1 Easy Example — cleaning input

Python
strip()
text = "   hello world   "
clean = text.strip()
print(clean)

Output: hello world

Time: O(n) • Space: O(n) (new string).

4.2 Medium Example — case conversion and comparison

Python
upper/lower
word = "Python"
if word.lower() == "python":
    print("Matched")

Output: Matched

4.3 Advanced Example — slicing to extract info

Python
substring extraction
email = "user@example.com"
name = email.split("@")[0]
# Note: split() is introduced lightly; later we expand.
print(name)

Output: user

Time: O(n) for scanning • Space: O(n) for parts.

8) Built-in Functions

  • len(s) returns number of characters.
  • str(x) converts values into strings.
Python
len()
word = "notebook"
print(len(word))

Output: 9

Time: O(1) or O(n) depending on implementation details; conceptually O(1). Space: O(1).

Alternative approach: manual counting (not recommended).

9) Methods

Common string methods

  • upper(), lower()
  • strip()
  • replace(old, new) (concept)
  • split(sep) (concept)
Python
replace() + split()
text = "one,two,three"

clean = text.replace(",", "-")
print(clean)

parts = text.split(",")
print(parts)

Output: one-two-three and then a list-like output from split.

Time: O(n) • Space: O(n).

Alternative approach: use loops to split manually (not needed unless learning).

15) Debugging Practice

Case 1: indexing off-by-one

Python
bug
s = "Python"
print(s[1:4])  # expected "yth"
print(s[1:3])  # buggy slice

Fix: end index is excluded. To include index 3, you need :4.

Case 2: trying to change a string

Python
bug
s = "cat"
s[0] = "b"
print(s)

Fix: strings are immutable. Build a new string using slicing.

Case 3: forgetting strip()

Python
bug
name = input("Enter name: ")
if name == "Alice":
    print("Hello Alice")

Fix: compare cleaned text: name = name.strip().

16) Common Errors

  • TypeError: mixing strings with numbers without conversion.
  • IndexError: using index out of range (like s[100]).
  • SyntaxError: missing quotes around string.
  • AttributeError: calling a method on a non-string value.

17) Best Practices

  • Use strip() on user input before comparisons.
  • Use slicing for substring tasks.
  • Prefer readable transformations: clean → transform → print.
  • When predicting output, remember: slicing end is excluded.
  • Don’t try to modify strings directly—create new strings.

18) Summary

You learned that strings are text values. You can access characters using indexing and parts using slicing. You used methods like upper(), lower(), and strip() to clean and transform text.
