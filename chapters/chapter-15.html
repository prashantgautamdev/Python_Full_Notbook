1) Introduction

Until now, your programs printed results to the screen. But many real programs need to save data to a file.

Real-life example

A notes app saves your text into a file. A log system writes events into a file.

2) Theory

2.1 open() and file modes

  • "r": read (default)
  • "w": write (creates or overwrites)
  • "a": append (creates if missing)

2.2 with open()

with ensures the file is closed automatically. This prevents resource leaks and locked files.

⚠ Warning

Writing with "w" overwrites existing contents. Use "a" when you want to keep old data.

2.3 reading patterns

  • read() reads the full file into a string.
  • readlines() reads lines into a list.
  • Looping over the file reads it line by line (memory-friendly).

3) Syntax

Python
read with with
with open("notes.txt", "r") as f:
    content = f.read()
print(content)
Python
write / append
# overwrite
with open("log.txt", "w") as f:
    f.write("Start\n")

# append
with open("log.txt", "a") as f:
    f.write("Next event\n")

Time complexity: O(file_size) • Space: depends on reading method.

Alternative: process line-by-line in a loop for large files.

4) Examples

4.1 Easy Example — write a file

Python
create + write
name = "Ravi"

with open("greeting.txt", "w") as f:
    f.write("Hello " + name + "\n")

print("Wrote greeting.txt")

Output: Wrote greeting.txt

4.2 Medium Example — read and count lines

Python
readlines + len
with open("greeting.txt", "r") as f:
    lines = f.readlines()

print("Line count:", len(lines))

4.3 Advanced Example — line-by-line processing

Python
loop over file
total_chars = 0

with open("greeting.txt", "r") as f:
    for line in f:
        total_chars += len(line)

print("Total characters:", total_chars)

6) Memory Visualization

read() loads the whole file into memory as one string. for line in f: keeps only one line at a time.

Method Memory use
read() O(file_size)
readlines() O(number_of_lines)
Line-by-line loop O(1) extra (plus output)

15) Debugging Practice

Case 1: FileNotFoundError

Python
bug
with open("missing.txt","r") as f:
    content = f.read()

Fix: create file with w or handle error with try/except.

Case 2: overwriting by mistake

Python
bug
with open("log.txt","w") as f:
    f.write("First\n")

# Later you expected append but used w again
with open("log.txt","w") as f:
    f.write("Second\n")

Fix: use "a" for append.

Case 3: forgetting newline

Python
bug
with open("notes.txt","a") as f:
    f.write("Note1")
    f.write("Note2")

Issue: Note1 and Note2 are stuck together.
Fix: add "\n" after each note.

16) Common Errors

  • FileNotFoundError: reading a missing file.
  • PermissionError: file cannot be accessed.
  • Wrong mode: using w when you wanted a.
  • EncodingError (concept): reading text with wrong encoding.

17) Best Practices

  • Use with open() always for files.
  • Choose correct mode: r, w, a.
  • For large files, process line by line.
  • Include newlines when writing lines.
  • Handle file errors with try/except if user input controls filenames.

18) Summary

File handling lets your Python program read and write data persistently. Use with open(...) to open safely. Read with read() or readlines(), and write with "w" or append with "a".
