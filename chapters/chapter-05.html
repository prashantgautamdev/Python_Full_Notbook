🎯 Learning Objectives

By the end of this chapter, you will be able to:

  • Use arithmetic operators like +, -, *, /.
  • Use comparison operators like ==, >, <=.
  • Use logical operators like and, or, not.
  • Predict outputs that involve operator results.

1) Introduction

After learning variables and types, the next question is: How do we combine values? That is the job of operators.

⭐ Remember

Arithmetic operators give numbers. Comparison operators give True or False. Logical operators combine booleans.

2) Theory

2.1 Arithmetic operators

  • + addition
  • - subtraction
  • * multiplication
  • / division (float result)
  • // floor division (integer-like result)
  • % remainder (modulo)
  • ** exponent (power)

2.2 Comparison operators

  • == equal to
  • != not equal
  • > greater than
  • < less than
  • >= greater or equal
  • <= less or equal

2.3 Logical operators

  • and True only if both sides are True
  • or True if at least one side is True
  • not flips True/False

⚠ Warning: = vs ==

= assigns a value. == compares for equality. Beginners often mix them and create logic bugs.

3) Syntax

Python
operators
a = 10
b = 3

print(a + b)   # 13
print(a - b)   # 7
print(a * b)   # 30
print(a / b)   # 3.333...
print(a // b)  # 3
print(a % b)   # 1
print(a ** 2)  # 100
Python
comparisons and logic
x = 7
y = 9

print(x == 7)   # True
print(x > y)    # False

print((x > 5) and (y < 10))  # True
print(not (x < 0))            # True

Time complexity: O(1) for single operations • Space: O(1) extra.

Alternative approach: use parentheses to control order and readability.

4) Examples

4.1 Easy Example — modulo for patterns

Python
odd/even using %
n = 9

if n % 2 == 0:
    print("Even")
else:
    print("Odd")

Output: Odd

Time: O(1) • Space: O(1)

Alternative approach: check n & 1 in bitwise logic (advanced).

4.2 Medium Example — comparison chains (concept)

Python
range check
score = 72

print(score >= 60 and score <= 80)

Output: True

Time: O(1) • Space: O(1)

Alternative approach: use chained comparisons: 60 <= score <= 80 (later referenced).

4.3 Advanced Example — mixing types (careful)

Python
string repetition
word = "ha"
print(word * 3)

Output: hahaha

⚠ Warning

Operators behave differently depending on types. * can mean multiplication (numbers) or repetition (strings).

Time: O(n) where n is output length • Space: O(n) for output string.

Alternative approach: prefer explicit conversions and comments when mixing operations.

7) Dry Run

Python
dry run
n = 8
print(n % 2)
print(n % 2 == 0)
Step Line Computed Result
1 n % 2 8 divided by 2 leaves remainder 0 0
2 n % 2 == 0 Compare remainder with 0 True

Time: O(1) • Space: O(1)

Alternative: use if n & 1 == 0 (advanced bitwise).

8) Built-in Functions

  • type(): helps verify results of operations.
  • bool(): converts values to boolean (careful: truthiness).
Python
type and bool
a = 5
b = a > 2
print(b)
print(type(b))

Output: True then

Time: O(1) • Space: O(1)

Alternative: skip type() once you are confident; still useful for learning.

9) Methods

Operators can work with strings. Methods help transform strings. Example: check length using len() and compare.

Python
string method + operator
word = "learn"
upper_word = word.upper()

print(upper_word == "LEARN")

Output: True

Time: O(n) for uppercase • Space: O(n) for new string

Alternative: use lower/upper based on your input rules.

15) Debugging Practice

Case 1: = instead of ==

Python
bug
x = 5
if x = 5:
    print("Equal")

Fix: use if x == 5:.

Case 2: wrong boolean logic

Python
bug
temp = 35
if temp > 30 or temp < 0:
    print("Hot")

Hint: decide if you meant “very hot” or “extreme hot/cold”.

Case 3: operator confusion

Python
bug
a = "5"
b = 2
print(a % b)

Fix: convert a to int first: print(int(a) % b).

16) Common Errors

  • SyntaxError: using = inside conditions instead of ==.
  • TypeError: using arithmetic operators on incompatible types.
  • Logic error: wrong use of and/or leading to wrong True/False.

17) Best Practices

  • Use parentheses in complex conditions.
  • Comment on non-obvious boolean logic.
  • Use == for comparisons, = for assignment.
  • Convert types before doing math.
  • Practice output prediction often.

18) Summary

Operators let you compute values and make decisions. Arithmetic operators produce numbers. Comparison operators produce booleans. Logical operators combine booleans. You also learned the classic = vs == warning.
